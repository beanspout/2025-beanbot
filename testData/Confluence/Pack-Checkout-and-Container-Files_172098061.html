BTSI Lab Systems Integration : Pack Checkout and Container Files

Created by Benjamin Fenner [X] on Jan 20, 2023

Problem

General issues with Pack Checkout

Solution

Confirming iTest is connected to the MRS Box

  • On BLINC1 Display press the Show MRS Box Status.
    • If MRS Box isn’t working
      • Check Power Harness and Firmware version.
      • Confirm ISO boxes are connected to the correct PC.
      • HS1 can signals may be nonactive/suspended check in VechicleSpy.
      • MRS Box may not being getting power. Look on the MRS Box in the white connections for a green light (Light is easy to miss).
      • CAN signals may be disconnected.
  • Powered on button is pressed but system faults.
    • Confirm that the cycler is on, isn’t knifed, has OK safety limits.
    • Is the power supply connected? Confirm that if the harness is connected and the iTest a 12V has feedback.
    • For Truck Packs.
      • Branch is old, or the MRS box is the wrong version which can be checked in the MRS Box Status display.
      • Look at the voltages on the SPI and Cycler and confirm that they match.
        • If not the CMUs may not be mapped correctly.
        • Swap MAC addresses – Press the 12V Button, Then the Run Manual Comms Button.
          • You’ll need to connect the manager, put the manager in standby, Set manager ACL, copy the MAC addresses and reorder them (Top half to the bottom). {Connect to Manager 5,0,1,60, Get Manager ACL 9, Set Manager ACL 8 (copy the list of MAC addresses and paste them into the cmd) enter, 6,0.
  • HV Wiring Polarity Fault
    • Ready the Cycler.
  • ADI_WIL_ERR_TIMEOUT Fault
    • Is the 12V Power Button On? (Green)
    • Use the VehicleSpy to resynch offset.
    • Unplug the all WeMes and reconnect them to a different USB port.
  • ACL/MAC Address(es) do not Match
    • Open Manual Comms.
      • Set Manager ACL 10, How many new ACLs, Index number (where the incorrect MAC address is in the list), Set Manager Mode 8, 2, 12, 11.
    • Confirming that Container Files match what’s already there
      • Before or after you flash container files you may want to confirm what the current firmware version is.
      • For BMS
        • Select the firmware version folder…
        • C:\Apps\ADT\iTest4.1\solution_common\EXECUTE\BRFM_Comm\v1.1.0\bms\bmsc-output
        • In the above folder path open the “adi_bms_defs.h” file in notpad.
          • Confirm that the ADI_BMS_VERSION and ADI_BMS)CONTAINER_FILE_CRC matches the values in the pinnacle tester.
