BTSI Lab Systems Integration : Cell Lab Gas Sensor Module (Ss1)

Created by Andrew White (C) [X], last updated by Trevor J Smith (C) [X] on Aug 24, 2022

Overview

The Gas sensor safety module (Ss1) for cell lab is a stripped down version of the same set of procedures used in the wider battery lab at Warren, MI. The goal of the module is to integrate up to three analog signals from a gas detection device into the cell lab iTest solution and continuously monitor these signals and shutdown the test stand if cell off-gassing occurs. The contacts for module development information and specifics are Jeff Poole and Andy McMichael.

If one of the sensors detects a hazardous value of any of the 3 defined gas sensors, there will be relays fed a volt signal (between 12V to 15V) that when tripped, will then feed to one of the 3 analog input channels on the Bitrode cycler that voltage value and will then be read by iTest and responded to accordingly. If the LEL sensor goes off, it is designed to shut down the whole lab, sound the alarms, and close the bay doors to close off air flow to the rest of the building. If the CO or malfunction sensors go off, it will simply shut down the test and the chamber.

Per Jeff Poole, the 3 signals should be coming through channels 1, 2, and 3 on the Bitrode cycler. As of 8/24/22, the connectors have yet to be created, tested, or have the actual channels connected to on the Bitrode be standardized, so this could be subject to change.Â 

Integration

There are three analog input channels set up as "digital" type signals for use in the module.

  • diSs1LELSensor - set up to be used as the 'lower explosion limit' sense
  • diSs1COSensor - set up to be used as the Carbon Monoxide sense
  • diSs1SensorMalFunc - set up to be used as a general communication fault, i.e. if this sense drops then gas sensor unit is malfunctioning


These input channels will need to be mapped to "Auxillary" input channels on the first Bitrode cycler for each stand. This can be achieved using the "Channel Mapping" feature for this module in Test Manager. The channels will read as analog input voltage signals, and are designed to trip certain faults if they read above 11 volts.

Testing

Once setup, the module will need to be tested. For each sense line, the signal should be manually interrupted for long enough to generate an iTest fault, or you can use an independent power supply to manually apply at least an 11V analog signal to each of the Bitrode analog channels we are monitoring*. Safety module channels can be monitored via iTest, and any irregular software behavior can be corrected by an LSIE or A&D application engineer.

*NOTE: EDIT THE SYSTEM CALCULATION TO TRIGGER DUMMY FLAGS INSTEAD OF THE ACTUAL LIMIT FLAGS, OTHERWISE YOU COULD SET OFF THE LIMIT FOR THE LEL SENSOR AND DISRUPT TESTS ON OTHER CHAMBERS.
