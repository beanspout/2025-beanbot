BTSI Lab Systems Integration : Upgrading Cell Lab Test Stands to iTest BTS

Created by Andrew White (C) [X], last updated by Trevor J Smith (C) [X] on Aug 31, 2022

n This guide is to be used by either members of the LSIE team or A&D Technology engineers in explaining and preparing them for upgrading test stands at the GM Battery Cell Lab from A&DT's BcTS software to iTest BTS.

An upgrade should take approximately 3 days to complete with another day for verification testing, not including the time needed for IT to upgrade the PC to Windows 10 if it hasn't already been done.

If you run into any errors or issues and need help, refer to the "Additional Notes to BTS Install Guide.txt" file attached to this post as well, there are some troubleshooting tips included for issues that came up during other BTS upgrades. To access it, click the paperclip icon next to the article title and find the file from the file explorer it opens from there.

Upgrade Steps

  1. Once a test stand has become available for upgrade it is important to contact Estes site IT to make sure the PC being upgraded is the latest hardware and that is is running Windows 10 OS. It is important to let IT know to save the data and files associated BcTS (file location will usually be C:\ADT\iTest3.4).
  2. Once the test stand PC has be returned/confirmed by site IT, you can begin the iTest BTS install by downloading and running the install executable and associated files from Z:\Patches\To_Deploy\iTestBTSInstall. You will need an iTest license key to finish the install, use the "SerialNumber" from the Config.ini file from one of the other BTS PCs when you do the install. It doesn't matter if more than 1 PC has the same install key as another, the license permissions will be sorted out when you obtain the ".v2c" file from Craig at A&D after sending him the ".c2v" file. When the SystemConfig window pops up, make sure to set iView to Enabled (this can be done after installation as well, refer to step 10). Any base solution can be created at this time, as a relevant solution will be cloned from Bitbucket later in the process.
  3. Once iTest BTS is installed you will need to update HASP license. To do that, open the ANDRus.exe and select 'Collect Information' to update an existing license. This will generate a ".c2v" file that will need to be sent to Craig Giraud (cgiraud@aanddtech.com) or Don Nutter (dnutter@aanddtech.com) at A&D Technology. They will send back a ".v2c" update file that should be applied in the second tab of the ANDRus.exe interface. Once the update is applied, the test stand will be able run Test Manager and the iTest interface.
  4. Install Digi Realport Serial Driver from Z:\_Manual_Installs\Digibox using Setup64.exe. This will allow multiple serial connections to Windows COM ports through an ethernet cable to a Digi serial hub. Select the device at 20.0.0.2, and create 4 new COM Ports. You can leave other install settings alone.
  5. Open Windows Device Manager and open "Multi-port serial adapters" and right-click and select "Properties" on the Digiport device driver. From the Advanced settings tab you can set the serial connection COM ports, baud rate, and other protocol settings. You will want to set the four serial connections to COM ports 2, 6, 7 and 8 to match the settings of the driver assigned to that port number, found under the particular Device settings in Test Manager.
  6. Install VisualCN from Z:\_Manual_Installs\Bitrode_Cal_Software following the instructions in the README.txt.
  7. Install SourceTree from Z:\Patches\To_Deploy\SourceTree_Install. Then find the appropriate test PC solution here: Z:\A_N_D_Folder\BcTS to BTS Upgrades ToDo, and unzip it in the iTest 4.1 folder.
  8. Open TestMgr.exe and edit/confirm the circuit settings to match the test stand you are setting up. 
  9. Start iTest and confirm all cycler and chamber drivers connect to the hardware.
  10. Validate the PC name and workstation name. Setup iView's connection to iCentral (Server: ice-batna-win.gm.com, Port: 443) and verify that the test stand is visible in iCentral. To setup the iView connection, if you didn't set iView to "Enabled" during installation, you can go to iTest4.1 > Execute > SystemConfig.exe and enable iView from there. Do not set the port number here. Instead, after it is enabled, find the iView icon in the taskbar tray in the bottom right of the desktop, right-click it, and click "Settings". After it opens, click "Configure LabCentral", and this is where you will enter the port number and server address.
  11. After connecting the PC to the iCentral server, make sure to change all relevant .ini settings to match the workstation ID so LabCentral can properly categorize the data coming from the PC. This includes vsSysID in Master.ini in $SUPPORTDIR and iCentralTestCellName in Config.ini in $SYSTEMDIR.Carefully copy and paste all additional settings in Config.ini from any of the other BTS test cell PCs, but be sure not to overwrite the correct setting for iCentralTestCellName.
  12. Load a set of schedule and battery specimen files for validation. Work with the Cell Lab engineers and technicians to secure test articles and time for brief (2-8 hour) Validation tests. Confirm that iTest has successfully logged all data and that iTest can upload this data to iCentral.
  13. All done!
