Methodology

To release a new python wheel or an update to existing python wheels, it is necessary to update the requirements.txt file in the Z:\Patches\To_Deploy\python3_wheels directory with the name and/or new version number.

The order in which the wheels must be listed in the requirements.txt file is determined by the components needed for each wheel.Â  For example, Pythonnet requires the numpy wheel to be installed prior to installing Pythonnet, so any new updates would require numpy to be installed/updated prior to Pythonnet.Â 

The cmd prompt will display which wheel dependencies are required during installation.Â  If a dependency isn't found, no python wheels will be updated or installed.Â  It's all or nothing.

How to Download Python Wheels

Go to the linked Pypi page above.Â  When updating the wheel, you need to verify the Python version on the left side of the page.

If the currently deployed version of Python is not supported by the wheel, select the Release history page and find the newest version of the wheel that supports the deployed Python version.

Go to the Download files page and select the file that matches the build of the PCs in the lab (currently x64).Â  Some wheels only have tar ball format, zip format, or are not PC build specific.Â 

How to Test Python Wheels

Put the downloaded wheel or source file into the distribution folder on your local drive, i.e. C:\Temp\To_Deploy\python3_wheels.Â  Open the requirement.txt file on your local drive and update the version number to match the new wheel's version.Â  Any additional wheels can be added to the end of the list to begin with.Â 

All wheels must be tested prior to making updates to the Z: drive wheelhouse, as the files go live when the patch script detects an update to the requirements.txt file.

Open the cmd prompt and run py -3 -m pip install -r C:\Temp\To_Deploy\python3_wheels\requirement.txt --no-index --find-links file:C:\Temp\To_Deploy\python3_wheels

Running this command will go through each wheel one at a time and start the install process.Â  If a wheel fails to install the cmd prompt will report the reasoning.Â  This could be due to a version mismatch, missing dependency, or in rare cases, the wheel file being unusable.

How to Troubleshoot Installations

The most common installation issue is a missing dependency.Â  The cmd error output will state if this is due to a version mismatch, i.e. numpy must be version X.X to support pythonnet, a file issue, i.e. the x64 version isn't downloaded or won't load, or the version number in the requirement.txt didn't match the file in the wheelhouse folder.

Version Mismatch

This is the most common issue encountered when installing new python wheels or updating existing wheels.Â  The specific dependency wheel should be listed in the error message by name and version number.Â  Frequently it will state the minimum required version, indicated by ">3.x."Â 

First, if the wheel the installation depends on is already in deployment, move the target wheel's entry in requirement.txt so it installs after the dependency is installed.Â  You will likely need to do this a few times if the wheel has multiple dependencies.Â 

File Issue

If the cmd error states that there isn't a compatible version that matches the requirement.txt it is likely due to the version of the computer not matching the python wheel.Â  You may notice that everything installs on TCWS but not on certain laptops.Â  To resolve this, from the python wheel's download page, select the x64 version, tar/zip source file, or the universal wheel file and add those to the distribution folder.
