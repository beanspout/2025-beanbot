Previous Discussion

Email sent on 7/8/2020 with the following points

With Cylance in mind, we know from past testing that Cylance has flagged:

  • The A&D Technologies folders and subprograms in the Program Files and Program Files (x86) directories
  • C:\ADT (and sub directories)
  • C:\Apps\ADT (and sub directories)
  • C:\Temp\To_Deploy folder
    • This is were we deploy our lab software from the shared drive prior to install on the PC
  • The Patches folder under \\ice-batna-win.gm.com\config drive (\\ice-batna-win.gm.com\config\Patches\) which contains:
    • The quarterly patch from A&D
    • AutoIT scripts that are used to deploy the patch and all assisting software (python wheels, Intrepid NeoVI software, Git, etc). 
  • The restart script in \\ice-batna-win.gm.com\config\Patches\Restart_Autologin\
    • This automatically reboots our PCs after 10 days of uptime, while ensuring the restart doesn’t occur when the system is on test


While many of these directories and tools have been whitelisted for Cylance in the past, there are constant changes to scripts, and things ends up getting reflagged and quarantined. 

The purpose of our scripts is to prevent our team to manually install everything on 150 PCs.

Is there a way to guarantee that these directories remain excluded from the Cylance scan, so we can move forward with the Win10 update and our automated software deployment? If so, do you have an hour or 2 to work with me on configuring out setup to ensure everything works?
