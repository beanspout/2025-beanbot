BTSI Lab Systems Integration : Pulse Utilization Tool

Created by James Jezewski, last updated on Jan 13, 2021

GM has purchased a tool from AVL called Pulse. Senior lab management uses it to track utilization across all of the labs.

The utilization codes we use in the battery lab are slightly different than what the rest of global propulsion uses, so the LSIE team needed to write a wrapper that would convert the codes and send them to the pulse server.

The IT contact for the tool is Brian A. Klotz <brian.a.klotz@gm.com> and he should be able to answer and address any questions you have directly related to pulse

All of the pulse conversion tool source code is hosted in this folder inside of the patch script: https://tslm-bitbucket.gm.com/projects/BTSILIB/repos/patch_script/browse/PulseUtilization

The idea of this script is that it will be run automatically during login of the FID on a computer and always be running.  It also gets re run when the patch script (https://tslm-bitbucket.gm.com/projects/BTSILIB/repos/patch_script/pull-requests/34/overview) is run during test start.


All of the test cells report to Pulse as you can see below:

As of 1/13/21, if a stand is showing a status code of 10000000000 it means the utilization script is not running, or there is some other problem that is preventing the stand from communicating with the pulse server
