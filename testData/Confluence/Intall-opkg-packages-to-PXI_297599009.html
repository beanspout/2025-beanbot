BTSI Lab Systems Integration : Intall opkg packages to PXI

Created by Collin Winans, last updated on Oct 14, 2024

The PXIs install linux packages through a tool called opkg. (It works similar to apt, if you are familiar with linux)

Unfortunately, opkg does not seem to work directly through the GMLAN. NI has a procedure to connect a separate computer to use as a gateway via MAX:

https://knowledge.ni.com/KnowledgeArticleDetails?id=kA03q000000wwuSCAQ&l=en-US


I got this to work on my laptop by connecting an ethernet cable from my laptop to the test site local switch and setting a static IP on my laptop of 172.10.101.2

Disconnect the PXI from the GMLAN network (pull the ethernet cable from the black patch panel) but leave it connected to the local site network switch. Then follow the instructions at the above link



For example, I installed the following to run diagnostics for hard drive/RAM errors:

For this station with the input/output errors, we are suspecting a disk or memory issue, so we would like to run some tests to check them:

From the terminal in the PXI (or remoting through ssh from a PC) please do:

1. 'opkg install memtester' and 'opkg install smartmontools'
2. run `smartctl --all /dev/nvme0`
3. For running memtester, it is probably best to run from safemode (you can direct the target to restart in safe mode from MAX) to test the most amount of memory. We would like to test it with most of the free memory and for a few interactions as arguments. (For more details on memtester: https://linux.die.net/man/8/memtester )- Note that it is recommended to run memtester with root privileges. You should aim close to the amount of free memory you are seeing, in MAX for example, if memtester cannot allocate all of it, it will reduce the amount and try again.
