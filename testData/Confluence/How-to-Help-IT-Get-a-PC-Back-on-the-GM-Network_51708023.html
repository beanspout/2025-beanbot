BTSI Lab Systems Integration : How to Help IT Get a PC Back on the GM Network

Created by Chelsea Flattery, last updated on Jun 02, 2020

If a PC is on the guest network, we need to work with Tony Barbour to get it back on the appropriate network. 

Step-by-step guide

  1. Put in the following IT ticket: https://itcatalog.gm.com/portal/shop/catalog/50d705be-0503-4399-8e5d-d160fcd6de5a
    a.  Select Private LAN Connection Issue
    b.  Paste the hostname and wait for it to populate (may take 10 seconds)
    c.  Fill out the required fields, if you haven't submitted an issue for the same TWCS, select No.
    d.  In Additional Details state the the PC is off the network

  2. Log in with ID that has admin rights
    Login: .\labadmin

  3. Go to search and enter: Services

  4. Once the Services window is open, right click and stop the Wired Autoconfig service
  5. In the toolbar, right click and choose Open network and internet settings

  6. Click Change adapter options (this will look slightly different on Windows 7 PCs)

  7. Right click and Disable the NIC that has the network connection (Usually named GM LAN)

  8. Wait 30 seconds and Enable the NIC

  9. Once the network connection shows corp.gm.com, right click and run ASIS
  10. Send Anthony (Tony) Barbour the IP or hostname of the PC so he can remote in and complete the reconfiguration
