BTSI Lab Systems Integration : Package Loader

Created by Collin Winans, last updated on Feb 27, 2023

The Package Loader ingests the test/site/DUT definition files and pushes them to the storage folder for the BTS server.


Install

You can install BTS Server (from NI Package Manager) onto a laptop for development, and it will also install the Package Loader. See instructions here:Â Install BTS Server on laptop

I attached a zip of the complete Package Loader folder below too, just in case you don't want to install the full server:

PackageLoader-2022-11-14.zip


Setup

To run the package loader for the production BTS server, set the Windows environment variable "BTS_StorageRoot" to "\\systemlink_storage.edc.nam.gm.com\gpd_nidevsuite0006683_data002\bts\BCIC\FileStorage"

For a local BTS server on a laptop, set the Windows environment variable "BTS_StorageRoot" to "C:\ProgramData\National Instruments\Battery Test System\FileStorage"


Run

Place the new/updated definition files into a folder (you could put them anywhere, but for this example, I will put them in "C:\bts-packages" )

Open a command window in the package loader folder (usually C:\Program Files\National Instruments\Battery Test System\PackageLoader )

To do a dry run to test the loader without pushing the changes to BTS, run the command:

BatteryTestPackageLoader.exe "C:\bts-packages" -d

This should run without errors. Depending on what is in the folder, you may see a different number packages that loaded


To run for real and upload the packages to the BTS server, remove the -d option from the above command:

BatteryTestPackageLoader.exe "C:\bts-packages"
