BTSI Lab Systems Integration : Failed Jenkins Build Debugging Steps

Created by Constantin Balan, last updated on Oct 18, 2022

NOTE: The Bitbucket Personal Access Token is attached to Constantin Balan's account and is set to expire on 17 Oct 2023. This token is what allows Jenkins to interact with Bitbucket.

Oh no, your Bitbucket pull request is showing the dreaded red exclamation point.

This means that the build of your branch failed on the Jenkins CI server. When this happens you can reach out to Constantin Balan if you have questions or concerns, but first you should try running through the following debugging steps.

Step 1: Go to the Jenkins build page

This step can be done in one of two ways. The first way would be to go to the Jenkins IP (You don't have to log in to view the build logs): Dashboard [Jenkins], then navigate to your branch, and then click on the most recent build which can be found in the picture below:

The second way would be to click on the red exclamation point from above which will bring you to a page of all of the builds on your branch pictured below:

From here you should be able to click on the blue text underneath "Build" and it will bring you to the build page.

Step 2: Navigating the build page

Once you've gotten to the most recent build page of your branch you should see the following:

On this page two of the options on the left will be useful for figuring out what caused the build to fail:
1) Pipeline Steps

               &

2) Console Output

The pipeline steps page will show you at which point in the process the build failed. This page is less detailed console output, but is still there for you to use if desired. Overall console output will have the same information pipeline steps has, but in more detail.

Step 3: Potential reasons for the build failing

There are multiple reasons why the build might fail. Some of those reasons may be related to something you changed/worked on for the branch, others are completely unrelated to any work you have done. First here are reasons the build because of the work you did on the branch.

Branch related build failures:

Jenkins Build Process

The current Jenkins testing set up is that the solution sanitizer will run first. If this fails, the solution common build will not run at all.


1) The solution sanitizer has made changes to your branch

If this is the case you will see the following message → "ERROR: Build failed due to sanitizer script changes". You will see which files were changed displayed in this format:

These files were changed by the sanitizer:  M Sequences/CellBal1.tsc

A way to check which lines of the file were changed would be by creating a copy of the file that was flagged, run the sanitizer on your computer locally, and then compare the two (This could be done via BeyondCompare).

2) The build of your branch has given Jenkins errors

If this is the case you will see the following message → "ERROR: Build 'X' has failed on 'pipeline-name'". You will see the errors that were thrown in iTest displayed in the console log.

3) Other

If you are getting errors other than the two mentioned above, reach out to Constantin Balan because there is likely a configuration/authentication issue that needs to be extinguished
