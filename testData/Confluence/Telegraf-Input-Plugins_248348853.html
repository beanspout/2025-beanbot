BTSI Lab Systems Integration : Telegraf Input Plugins

Created by Constantin Balan on Feb 05, 2024

This page is an overview of all of the input data plugins Telegraf is pulling/is planned on pulling from the lab PCs. Inputs in Telegraf are fields sourced from different tools on the computer that aggregate the data, then push it off into an output source (in our case, InfluxDB). The ones included below are plugins that we currently have or plan on having in the near future in the Telegraf.conf file.

Telegraf Plugins

The input plugins below are open source and compile their fields from Windows Management Instrumentation.

inputs.cpu

The cpu plugin gather metrics on the system CPUs.

inputs.disk

The disk input plugin gathers metrics about disk usage.

inputs.internal

The internal plugin collects metrics about the telegraf agent itself.

inputs.net

This plugin gathers metrics about network interface and protocol usage (Linux only).

inputs.net_response

The input plugin test UDP/TCP connections response time and can optionally verify text in the response.

inputs.netstat

This plugin collects TCP connections state and UDP socket counts by using lsof.

inputs.win_perf_counters

This plugin is used to read Performance Counters on Windows operating systems.

The configuration is parsed and then tested for validity, such as whether the Object, Instance and Counter exist on Telegraf startup.

This consists of multiple custom measurements/objects that were created years ago. This is subject to change & clean up.


Telegraf Plugins to be implemented (As of 2/5/2024)

inputs.system

The system plugin gathers general stats on system load, uptime, and number of users logged in. It is similar to the unix uptime command.

inputs.mem

The mem plugin collects system memory metrics.

For a more complete explanation of the difference between used and actual_used RAM, see Linux ate my ram
