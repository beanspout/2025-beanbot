BTSI Lab Systems Integration : MAB Rescue

Created by Collin Winans on Mar 28, 2023

If the PXI chassis is taking an IP address in the range 26.X.X.X then it is not authenticated on the network.

The first time it is connected to the network, it will need to be added to the MAC Address Bypass (MAB) system. I usually send a batch of MAC addresses to Joe Juricic, but you could technically submit an IT ticket here: https://itcatalog.gm.com/portal/shop/catalog/b565f986-b3b3-4c77-948b-f22a7cde3faf (it's not a fast process, sometimes it takes several weeks to complete)

For MAB v1, any time the device disconnects from the network (e.g. a reboot) it will need to be "rescued." Eventually we are supposed to move to MAB v2, which can auto-rescue and won't require the following steps. For now we are stuck with this process.


Rescuing the MAB will create a short time window where the device is allowed to connect.

https://mabrescue.cpi.gm.com/

Enter the MAC address for the PXI into the first window (you can get the MAC from the NI Max software on the HMI PC):

If it is not in the MAB system then you will get an error and you need to add it through an IT ticket: https://itcatalog.gm.com/portal/shop/catalog/b565f986-b3b3-4c77-948b-f22a7cde3faf

If the device is in MAB then you will see the following window:

Select a time duration, then click rescue. This will start the time to allow reconnecting to the network.

Reboot the PXI, it should connect and get a new IP address in the 10.X.X.X range

Congratulations, you have now rescued a MAB
