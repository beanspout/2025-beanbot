BTSI Lab Systems Integration : Creating git tags for master branch releases

Created by Collin Winans on Feb 24, 2021

When we release a new version of the master branch, we also create a tag with the version number.

Step-by-step guide

  1. Complete the Pull Request
  2. In SourceTree, right click on the merge commit and select "Tag..."
  3. Type the tag name in the "Tag Name" box
  4. Check "Push tag" (to origin)
  5. Make sure "Lightweight tag" is NOT checked. Note, some git clients will default to using lightweight tags, which are not compatible with our branch tracking scripts
