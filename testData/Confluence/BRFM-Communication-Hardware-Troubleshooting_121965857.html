BTSI Lab Systems Integration : BRFM Communication Hardware Troubleshooting

Created by Rodolfo Rodriguez (He/him/his), last updated on Jan 31, 2022

The purpose of the this page is provide advice for troubleshooting common issues with the Battery Radio Frequency Module (BRFM). 

Background on Devices:

A Cheetah SPI host adapter (sometime referred to as Cheetah box) is required in order to communicate with BRFM using these scripts.
A Cheetah SPI host adapter is essentially a SPI to USB converter. It allows the PC to be able to transmit or receive serial
data via SPI protocol without interfacing with low-level codes. See https://www.totalphase.com/products/cheetah-spi/ for more info on Cheetah device.

The WeMe is a USB extender that extends the connection of a USB device up to 100ft. This is used to bridge the large physical distance between the test stand and Cheetah box. More information about the WeMe can be found here: Amazon link

Hardware Troubleshooting:

Some of the common faults related to hardware issues are the following:

"Failed to detect the presence of Cheetah connection!"

If the Cheetah box is not being recognized, perform the following:

  1. Check physical connections from the PC to the Cheetah box. Verify that the WeMes, ethernet cables, USB cables, and power supply for the WeMe are connected correctly.
  2. Check that the WeMes are working properly. All three lights should be on and the green light should be flashing. (Figure 1 and Figure 2)
  3. Check that the moving the cables does not cause the WeMe to lose connection. If faulty ethernet cable can disrupt the connection to the Cheetah box.
  4. Retry connecting to the cheetah box by performing the following:
    1. Go to the VRFM or BRFM Pinnacle Tester in and run manual communication. (Figure 3) Ensure that only one window of the Pinnacle Tester is open at time. (Figure 4)
    2. Enter 2 to "Get Cheetah device info".
  5. Possible outcomes:
    1. "Number of Cheetah Devices Connected: 0" (Figure 5) - Verify that the IO rack is enabled in iTest and the rack is powered ON. Try disconnecting and reconnecting the WeMes and unplugging and plugging the USB cables and power supply. Also try connecting the sender WeMe to a different USB port on the PC. While it has not been confirmed whether USB 2.0 or 3.0 makes a difference, this has sometimes helped resolve connection issues.
    2. "Unable to grab a Cheetah handle; The device is busy." (Figure 6) - Verify that there are no other instances of the Pinnacle Tester open. If you tried once already to Get Cheetah device info, it may show up as busy. Close the Pinnacle Tester and try again.
  6. If none of these solutions work, try replacing the WeMe sender and receiver, one at a time, with a new one. Sometimes the WeMes are not reliable despite the lights flashing correctly.
