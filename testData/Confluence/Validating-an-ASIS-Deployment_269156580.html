BTSI Lab Systems Integration : Validating an ASIS Deployment

Created by Chelsea Flattery on Apr 30, 2024

If you are utilizing a PC with a label indicating that it is an ASIS Pilot PC and that PC is listed in the ASIS Pilot PC Status Teams channel, perform the steps below to validate the ASIS pilot as pass or fail.

Step-by-step guide

Make sure all programs are closed before performing any steps below

  1. Reboot the PC, this will trigger ASIS to run and reset any network connectivity issues that could present as a false failure in deployment
  2. Wait for the ASIS window to appear and for the blue Deployments Complete text to display at the bottom of the display window
  3. If the PC reboots prior to deployments completing, ASIS will reopen when you log back in. It may continue running deployments, let it finish all processes before continuing.
  4. Open SourceTree and click Fetch, only check the top check box, and click OK.
  5. Click the red iTest Test Manager launcher icon on the desktop, this will test the functionality of Git, connection to Bitbucket and functionality of AutoIT in one fell swoop
  6. Click Skip Git Update and Open Test Manager and then allow Test Manager to open
  7. Click File and then Save All and Restart to open iTest Console
  8. Allow the system to initialize. Select the hardware setup that is applicable to the test stand state
  9. Open the Main display if necessary and click Update/View Notify Users
    ,
  10. Add yourself as a Requestor, this will test the functionality of Python and the iTest interface
  11. Remove yourself from the Requestor field and close iTest
