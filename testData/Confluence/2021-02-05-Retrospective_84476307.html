Retrospective

We ended up running this sprint more like Kanban instead of Scrum.

Not all items were assigned to individuals at the start of the sprint.


What did we do well?

  • We are working together to learn about iTest and the sprint process.
  • We have had a chance to work on fault improvements!
  • Open up the door for productive brainstorming.
  • Was able to learn a lot about fault handing.
  • Learning about the "guts" of iTest.

What should we have done better?

  • Some task are still not complete or are being reviewed.
  • If tasks aren't assigned to specific people, then a lot of tasks sit around.
  • It would have been better with more clear communication about grabbing vs assigning tasks.
  • Needed more time for sprint planning.
    • Some items were duplicated.
    • Some items ended up going in a unexpected direction.

Actions

  • Paul Rabahy Send out sprint kickoff notes by Tuesday morning so that people have time to review them for Wednesday kickoff meeting.
