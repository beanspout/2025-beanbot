BTSI Lab Systems Integration : Test Engineer Tells You Their Stand isn't Showing Up on Grafana

Created by Chelsea Flattery, last updated by Constantin Balan on Sep 13, 2022

So there's four actors involved here:

Test Stand (iTest)

telegraf

influxdb/iCentral datasource

Grafana

(Image is accurate as of 9/13/2022)

The test stand runs telegraf as a service, which will automatically send PC Stats like uptime, RAM, disk space. 

Once iTest goes through SysInit.TSC the SysMetricsStart and SysInfluxStart scripts will use telegraf to send data to influxdb or Grafana will pull from iCentral.

Influxdb sends data points to Grafana, which lives on a server in the data center, so Grafana can visualize the data points in graph form.

iCentral is the existing database for all of the iTest channels included in The Master Names List. iCentral has a plugin to send this data to Grafana in the form of data points.


What this means is, there are 4 - 5 points of possible failure that all react in different ways.

Failure Points:

Test Stands, if not running SysMetricsWD (watch dog) and SysInfluxWD will not send any test data, only system stats through telegraf

Telegraf could have been stopped or some other act of computer gremlins which would make it send nothing, no stats, no data points

(Currently not being used) Influxdb can get restarted by GM IT's ASIS deployment system triggering a shutdown or it can take in so much data that it starts to hiccup and will only send partial data points to Grafana

iCentral's Upload drive could be full, preventing all data from being uploaded to iCentral.

Grafana could be in an IT maintenance window (https://appdashboard.cpi.gm.com/) or something happens at the data center, causing it to stop working.


How do we know something doesn't work?

Test Stand - The most tricky.. You'll know this isn't working when a Test Engineer says that iTest is running and none of their data is in Grafana. If the stand is sending System Stats, you know that telegraf is working but SysMetrics and SysInflux watch dogs are not working, i.e. no iTest test data can be collected.

Telegraf - Test Engineer complains that there is no data in Grafana and the System Stats are not displaying.

(Not currently active) Influxdb - Grafana is running but there are error messages like this:

If you are seeing this message, you're using the wrong dashboards or the wrong Grafana website.  Go to Dashboards - Manage - iCentral - iCentral Quick Overview

iCentral - If the dashboards are loading but not displaying any information, check the Upload drive

Grafana: The website won't load. At all. You may also see something like this if the server needs to be rebooted:


So how do we fix it?

Test Stand - Emad created a fix for this in addition to a message to reboot the PC if there has been a lot of uptime between ending one test and starting another.  If all else fails, you can manually restart them by sending mailslots to the DataEngine: "EXECUTE SysInfluxStart" and "EXECUTE SysMetricsStart"

Telegraf - This can be restarted while iTest is running, it won't hurt anything. Open cmd as an administrator and send "net start telegraf"

If it tells you that telegraf is running, send "net stop telegraf" wait a few seconds then send "net start telegraf"

(Not currently active) Influx - https://tslm-confluence.gm.com/display/BTSILSIE/Make+Influx+Work+Again

iCentral - How to Fix the iCentral Grafana plugin

Grafana - When IT hosted things stop working, Grafana or the camera Relay Server, you message George Baseski and ask him to restart whichever application is down
