BTSI Lab Systems Integration : How to Move Service Desk Issues to Jira Software

Created by Chelsea Flattery on Aug 12, 2020

Scenario: Test Engineer puts in a Service Desk ticket and we need to relate it to a Release branch or Sprint planning

Step-by-step guide

From the Service Desk issue:

  1. Click the More drop-down menu
  2. Click Create linked issue

 

  1. Change the Project to BTSI Common iTest Solution
  2. Select any applicable Issue type (this can be changed later)
  3. Choose any applicable option for how to created issue relates to the service desk request
  4. Add any additional linked issues, if applicable
  5. You may choose to copy attachments and links from the original service desk request
  6. Click Create

NOTE: If you have copied attachments and links:

The following error message will appear, ignore it and no not click Create a second time, the issue has been successfully created.  Close the window, accept the prompt to close without saving, and complete next steps outlined below.


Navigate to the BTSI Common iTest Solution page (Jira Software) and find your issue, it will have the same name as the Service Desk issue, but a different number.

Note the different space name and icon to make sure you're in Jira Software.

Depending on the options you selected when creating the linked issue, attachments will be present in the new issue.

  1. Click on the pencil icon next to Fix Version/s
  2. Use the drop-down menu to select the Version


You can also add this information by clicking Edit on the issue page and adding the Fix Version/s and any other additional information
