BTSI Lab Systems Integration : How to Deploy a New Patch

Created by Chelsea Flattery, last updated on Oct 11, 2022

There are two deployment strategies for deploying a new patch, one is for code that doesn't require a full patch script run and the other is for code that requires the patch script to run in full. Regardless of patching strategy, this process must be done both for Warren and Shanghai, as they use different Config drives.

Partial Patch Deployment

When would I do this?

If your code both needs to go live immediately (like a hotfix) and/or will run between patch script line 1 and approx. line 550

If your code does not need to go live immediately but instead will be activated by running the patch in debug mode (holding F9 while the script launches)

Full Patch Deployment

When would I do this?

If your code occurs after approx. line 550

If your code includes files that need to be downloaded from the Z: drive to the local test stand

Step-by-step guide

After validating your code and merging to master, pull the latest master onto your local machine.Â  The files in your patch_script repo will be updated to reflect the latest master and are ready to deploy.

Partial Patch Deployment
  1. Navigate to Z:\Patches\patch_script
  2. Delete Updater_old.au3, Installers_old.au3, HelperFunctions_old.au3, and InstallVehicleSpy_old.au3
  3. Rename Updater.au3, Installers.au3, HelperFunctions.au3, and InstallVehicleSpy.au3 to add _old.au3 to the end of each filename
  4. Paste the new Updater.au3, Installers.au3, HelperFunctions.au3, and InstallVehicleSpy.au3 files
  5. Repeat this process for China by remoting into a PC in the Shanghai lab and following the steps above, copying the patch script files from US Config drive to the China Config drive (it will be too slow to copy from your machine to the China Config drive over Remote Desktop)
Full Patch Deployment
  1. Create a newly dated folder in Z:\Patches, i.e. 2022-10-11 for both US and China
  2. Paste the iTest version folders from the previous patch (if unchanged) or from Z:\Patches\NextPatch (if different) into the new folder
  3. If you are releasing an AND patch to iTest, unzip the file and place it in the AND_Config folder
  4. Remote into a PC in Shanghai and repeat this process, copying from US Config drive to China Config drive
  5. Complete the steps above for both US and China
  6. Open Z:\Patches\Inventory.ini and replace the PatchVersion variable with the new patch date
  7. Open T:\Lab_Config\Patch_Info.ini and replace the PatchVersion variable with the new patch date

As soon as you change the date and save Patch_Info.ini the script goes live and the patch script will force a full run of the script because it detects a change in the patch date.
