BTSI Lab Systems Integration : How to Use/Navigate the Jenkins Website & CI/CD Pipeline

Created by Constantin Balan, last updated on Apr 26, 2023

This is an overview page of how to navigate the Jenkins server and a general explanation as to what Jenkins is and how it's used. It is currently set up on the labminder PC, but will migrated to Kubernetes by the end of 2022.

Current Jenkins Log-in URL: Sign in [Jenkins]

Welcome to Jenkins! Above is the log in page for our locally hosted Jenkins instance. If you go directly to the url:port without logging in or being logged in you will only have read access. So logging in allows you to run tests and edit settings.

Dashboard Overview

After logging in you will be brought to the dashboard page (shown below).

On the left hand side you will see the different pages you can get to from the dashboard. They are as follows:

New Item: Clicking on this will allow you to create a new project. You will be given a couple of different presets to choose from such as Freestyle project, Pipeline, and External Job. In the future if you are trying to make a new project, this is where it would be created from.

People: This will show you a list of all of the active accounts associated with that Jenkins instance.

Build History: This will display all of the different builds in chronological order. Clicking on a build on this screen will allow you to expand it and allow you to see the details of the build.

Project Relationship: If you have multiple projects that depend on each other, you can use this page to track which build of one project relates to the other. For our sake (at least currently), we won't have any use of this page.

Check File Fingerprint: This page will help you figure out what versioning .jar files are.

Manage Jenkins: This is the brains of the Jenkins machine. Configuring things such as file paths, environment variables, plugins, etc. are modifiable from here. 

Try not to edit things on this page unless you know what you're doing because it can potentially mess up other running projects. 


My Views: This page will show all of the items your user has access to. For now we only have the solution_common pipeline, so that's all that will show up.

Open Blue Ocean: Blue Ocean is a nice UI that is integrated with Jenkins and managing its projects. Unfortunately it's buggy and doesn't work with our repos, so this tab is useless.

Lockable Resources: This page is where you will find any resources (such as printers, phones, computers, etc.) you have reserved to run specific jobs/builds. We don't currently implement anything in this regard, but may possibly do so in the future.
New View: This page will allow you to create a customized view in the scenario where you have a lot of different projects running. We don't have multiple projects, so this isn't of any use yet.

Solution Common Multibranch Pipeline Overview

If you select the Solution Common Multibranch Pipeline from the view, you will be brought to the overview of the project (shown below):

On the left hand side you will see the different pages you can get to from the project overview. They are as follows:

Up: This will just bring you back to the Jenkins dashboard

Status: This shows you the status of all of the branches that the pipeline is running or has been run on. The image above shows what that looks like.

Configure: The configure page is where you can modify different aspects of the pipeline such as what repository it's pulling from, what will trigger a build, and other properties.

Reminder: Multibranch Pipelines are single repository entities, therefore it can only be hooked up to one repository.

Just like with above, try not to edit this unless you know what you're doing because it can potentially break something with the pipeline.


Scan Multibranch Pipeline Now: Clicking this button will scan all of the repository branches and check for a specified Jenkinsfile. A branch needs a Jenkinsfile (Jenkinsfiles are what Jenkins uses in a multibranch pipeline to determine what steps to take on the branch. It's where the instructions for Jenkins are stored) to do anything, so any branches with no jenkinsfile will not be worked on.
Scan Multibranch Pipeline Log: This is where you can see the results of the scan.

Note: Keep in mind that Jenkins will search for a jenkinsfile with the exact named it's given. So if you have a jenkinsfile named "Jenkinsfile.groovy", telling Jenkins to look for a jenkinsfiles with the name "Jenkinsfile" will cause it to not find the former. So make sure to include the entire file name, extension and all, if there is a jenkinsfile but Jenkins still cannot recognize it


Multibranch Pipeline Events: This page will show you the events (e.g. PR created) on the pipeline
Delete Multibranch Pipeline: Self-explanatory. Please don't do this unless we have a very good reason

People: This page shows all of the users that have made changes/have access to editing the pipeline

Build History: Identical to the dashboard "Build History" page, but only pertains to the multibranch pipeline.

Project Relationship: Identical to the dashboard "Project Relationship" page.

Check File Fingerprint: Identical to the dashboard "Check File Fingerprint" page.

Browse Repository: Clicking on this will re-direct you to the Bitbucket repository that is linked to the pipeline (In our case, solution_common).

Open Blue Ocean: Identical to the dashboard "Open Blue Ocean" page.

Rename: This will allow you to rename the multibranch pipeline if so desired.

Pipeline Syntax: This page will allow you to create snippets of pipeline appropriate syntax. For example, if you want to run a batch script and don't know how to specify that for Jenkins, you can select "bat" from the sample steps, and then enter the batch file you would like to run.

Credentials: This page shows you all of the credentials (e.g. SSH keys) associated with Jenkins. 

New View: Identical to the dashboard "New View" page.
