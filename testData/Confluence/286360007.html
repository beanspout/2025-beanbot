BTSI Lab Systems Integration : List of Software & Tools on the LSIE Team

Created by Constantin Balan, last updated on Sep 04, 2024

Below is a list of software and custom tools made by and used by the LSIE teams and in some cases the lab technicians.

Software

Ansible: Ansible is an automation software that can be used to automate most if not every manual task on Windows and Linux servers. Ansible runs an open-source option (currently being implemented in the battery lab) and a paid version called Ansible Automation Platform (AAP). AAP is currently in use in other departments at GM, and the idea is eventually we will get onboarded into their sphere of support once they begin supporting Windows servers.

AnywhereUSB: Digi's AnywhereUSB is a device that allows for multi-port USB connectivity to devices.

Bitbucket: Bitbucket is a source control platform owned by Atlassian. We have multiple repositories hosted on Bitbucket for our various pieces of software. Ex. iTest solution common, the patch script, the iTest launcher, etc.

Confluence: Confluence is a knowledge base tool owned by Atlassian. We use Confluence for documentation purposes on our software, tools, and processes.

Digibox: 

Git/SCM: Git is a version control system used to keep, for a lack of a better word, backups of code. We use git to keep versions of our software that we publish to Bitbucket. The LSIE tends to use Git GUIs like Sourcetree to manage gitflow. There are other tools you can use Git in such as Git Tortoise and Git Bash.

Grafana: Grafana is a data visualization software. It pulls in data from data sources, in our case InfluxDB. It then can be configured to display data over time in a multitude of different graphs.

iCentral: iCentral is data aggregation and system monitoring tool for all TCWS machines running iView. Test stands will periodically publish test data to iCentral drives. There is also functionality to see a desktop view of each connected TCWS machine.

iView: iView is a client-side software that runs along with iTest. It's used to allow iCentral to "see" the client iView is running on. It then sends pertinent data to iCentral.

InfluxDB: InfluxDB is a time-series database that is used to collect iTest data (via custom Python scripts), system stats, and specified logs (Grafana Influx engine, patch script, etc.). It receives these data points from a collection system like Telegraf or custom Python scripts and then stores the data on the Influx servers in buckets. These data points are then queried on Grafana and displayed on dashboards. 

Jenkins: Jenkins is an open source CI/CD software that is hosted by the GM Jenkins team. It is being decommissioned in Q4 2024. 

Jira/Jira Service Desk: Jira is a bug tracking/issue tracking software owned by Atlassian. Service Desk is a customer facing aspect of Jira that allows users to enter in tickets/issues that can automatically be filtered. We use Jira for keeping track of our work based on the projects we assign them to.

PagerDuty: PagerDuty is an alerting software that sends pages to phone numbers, e-mails, and so on. It has an escalation system that allows alerts to be escalated to higher ups when the initial pagee misses the page. Currently in the lab we have a rotating duo of first and second shift individuals on a weekly basis.

PostgreSQL Database: PostgreSQL is an open source relational database management system. The LSIE team has test/staging/prod instances set up with several tables already in prod such as git branch info, pc list info, and calibration info.

Sourcetree: Sourcetree is a Git Desktop GUI owned by Atlassian. It is used to visualize branch flow and let you visualize the code inside of commits.

Telegraf: Telegraf is a client-side data collection service. It can be configured via a .conf file that tells it what inputs and outputs to pull data in and send data to.

Custom Tools & Languages

AutoIT-Based Patch Script: The patch script is a custom-built piece of software written in the Windows AutoIT automation language. It's currently used to set paths/values and install/update software relating to and supporting iTest.

iTest Launcher GUI: The iTest GUI is a custom GUI that launches after the AutoIT patch script runs. It's used to select a branch that will be loaded into iTest to allow custom tests to be run. It currently has two version. The old version (white box with a large scrollable selection of branches) and a newer un-implemented version (Smaller/quicker version that launches in dark mode, written in Golang). Once the test cells are fitted with dedicated graphics cards, the new version will continue to be tested and rolled out once in a satisfactory state.

Python Support Scripts: We have a multitude of python support scripts. Varying from supporting different iTest functionality to batching custom iTest data points to InfluxDB.

TCWS Site Logger: The TCWS Site Logger is a tool is being currently developed. It will run client-side on the TCWS machines and will keep a running log of activity done on the test cells. The activity will be logged manually by users, and will then be ingested by Telegraf, sent to InfluxDB, and displayed on a Grafana dashboard.
