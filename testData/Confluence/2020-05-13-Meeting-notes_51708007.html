Meeting Agenda/Discussion Points:


How can we improve the current PQMS process with regards to communication between stakeholders

Hard to navigate, lots of attaching Word documents that have embedded documents within them

No issue filtering in PQMS

Doesn't connect to an API for requirements, specs, configuration

No way to post progress or updates on the original PQMS

Communication would be improved if it occurred within a system that is being used everyday to log work

Too many embedded documents


What would PQMS look like within Jira

Jira and PagerDuty can be integrated so faults can be posted automatically to the Jira issue (this needs the PD API to function)



(How) Can we implement the use of statuses within Jira 

Improvement opportunity: When we start a test or a test faults, have a notification posted within Jira Service Desk or PagerDuty

Could PD integrate with Jira to post the update?

When you enter the test data, add the link to the service desk ticket, this could grab the issue participants

It would be replacement for the orange window on who to notify, then this pulls escalation policy


Proposed approval by the requester
