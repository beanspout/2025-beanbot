BTSI Lab Systems Integration : Common faults for truck pack communication

Created by Nguyen Phan on Nov 22, 2022


BDSB current sensors not mapping correctly during the Pack A pulse

  • Check the order of MAC Address: (truck pack: 1-12 for UPPER, 13-24 for LOWER)
  • Check the cycler is in independent mode (remote mode) and interlock close
  • Monitoring the cycler current (75A) and each half of the pack current

 

IsoSPI lines might reverse

  • Swap isoSPI lines connectors


BLINC1_ET1B_PackCheckout: The CMUs did not map correctly during the Pack A pulse. Please check the module MAC addresses

  • Check the value of varBLINC1PulseCellDiff  in PackCheckOutTrans and change the value of vBLINC1_ET1B_CheckoutPulseThresh (not vBLINC1_ET1B_CheckoutNoPlsThresh)
  • Swap the MAC addresses between upper and lower half 


BLINC1_ET1B_PackCheckout: Check MRS box connectors and HV wiring polarity! vPps1In:0 vBLINC1:398.6

  • Check MRS hardness. Make sure MRS box message come to CanNeo
  • Make sure contactor close
  • Make sure cycler initialized
  • Low voltage to chassis ground
  • Check two 12V power of the MRS box (use internal power supply instead of Accopian due to common ground)


Aux7 has mismatching checksum values! (FFFFF in PMS data) /  PMS B3TA exceeded consecutive packet error threshold! Data may be stale

  • PMS wrong container
  • BDSB doesn’t have power, check chassis ground
