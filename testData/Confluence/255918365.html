BTSI Lab Systems Integration : How to Test a New Patch/New Release

Created by Chelsea Flattery on Mar 06, 2024

Whether you are testing a change that will run during a partial patch run or a full release run, there are multiple steps to perform.

Testing Modifications to the Patch Script to Simulate Deployment

After verifying that your patch script code works on your local machine, both in normal and debug mode (holding F9 to trigger), you are ready to move your scripts into the patch script beta script folder.

  1. Navigate to Z:\Patches\patch_script\patch_scriptbetascripts\ and create a folder either with the Jira ticket number or a common name for what you are releasing.
  2. Copy the BatchGotAdminForBeta.bat script into that folder
  3. Open in VS Code or other editing software and modify the highlighted file path to match the folder you created in Step 1.
  4. Save file
  5. With your working branch pulled, copy the following files from your local repo into the folder created in Step 1.:
    1. HelperFunctions.au3
    2. Installers.au3
    3. InstallVehicleSpy.au3
    4. Updater.au3
  6. Double-click the BatchGotAdminForBeta.bat file to test your code

Testing a New Release

If you are testing a new release in conjunction with the guide here: How to Deploy a New Patch complete steps 1-5 from above.

  1. Double-click BatchGotAdminForBeta.bat and hold the F9 key to trigger Debug Mode
  2. When you see the following popup, click No
  3. When the folder selection window appears, click the dated release folder that you are testing only once, i.e. 2024-03-13
  4. Click Select Folder

How to Test a New Patch and Open iTest

If you are testing a new patch and then need to open iTest, it is important to make sure the patch script that is live in deployment doesn't overwrite any changes you created.

To do this, you must disable patches in iTest.

  1. Click the Windows icon/press the Windows key to open the program menu
  2. Open the A&D Technology folder
  3. Click Test Manager. Opening it this way prevents the patch script from running prior to you disabling it.
  4. With Test Manager open, click to expand the Channels folder and select Internal Variables
  5. Set flgSysDisablePatch to 1 and click Save All and Restart to launch iTest.
