BTSI Lab Systems Integration : How to Configure IVC Cameras

Created by Chelsea Flattery, last updated on Dec 08, 2022

Reasons for configuring the IVC cameras directly and not using the IVC Relay Server:

  • Camera is new and needs to be configured for the first time
  • An IP address must be changed on a previously configured camera
  • The 802.1x credentials need to be renewed
  • Camera overlays need to be changed

You must have access to the SDF VPN to perform some of these actions. See this article for who has access: How to Get Access Through the SDF

Initial camera setup guide

For the love of your own sanity, perform this step prior to connecting the camera to the ceiling. 

What you need:

  • The IP address provided by IVC, otherwise you'll need WireShark
  • PoE Injector (currently in Chelsea's drawer 13)
  • 2 Ethernet cords >2 feet
  • Power source (switch closet A8's outlet is above the server stack so be or bring someone who is at least 5'9")
  • A laptop

Setup at bench

  1. Disconnect laptop from WiFi
  2. Connect PoE Injector to a power source
    1. If the light doesn't turn on, choose another outlet (especially if you are in closet A8 where all but one outlet is disconnected)
  3. Plug ethernet cord connected to laptop into the LAN port on the PoE injector
  4. Plug ethernet cord connected to the camera into the PoE port on the PoE injector
  5. While the camera power cycles, open Network Connections on the laptop
  6. Right-click on the ethernet that is showing as connected or that you are connecting with and select Properties
  7. Double-click the Internet Protocol Version 4 (TCP/IPv4) connection
  8. With the Use the following IP address radio button selected,
    1. Manually set the IP address to an IP that is within range of the camera's IP
      1. i.e. If the camera's current factory default IP is 192.168.0.140, set the laptop to 192.168.0.139
    2. Set the Subnet mask to the same domain as the camera
      1. This is usually 255.255.255.0
  9. Click Ok on both windows to return to Network Connections
  10. Wait for the selected Ethernet to say Connected, this can take some time, especially if the camera isn't done power cycling
  11. Open an internet browser
  12. Enter the IP address of the camera you are trying to connect to
  13. At the camera login page enter the default camera credentials (capitalization matters)
    1. Find the credentials Z:\Software Team\_LSIESystems.ini
    2. If neither works, email IVC Support with the camera model type and serial number
  14. Select Setting from the top menu bar
  15. Click the Network drop-down menu and select 802.1x
  16. Click the Enable checkbox
  17. Set Authentication to PEAP
  18. Enter credentials listed for SI Account in Z;\Software Team\_LSIESystems.ini
  19. Vivotek cameras need a CA certificate
  20. Note to Chelsea: Document how to do this when you get the Vivotek camera on the bench
  21. Click Save
  22. The camera will display a configuration popup and will reboot (if it doesn't.... 802.1x doesn't work on the camera)
  23. Once the camera reboots, log in again
  24. Click Settings from the top menu bar
  25. Click the Network drop-down and select TPC/IP (this must be done second or you're going to have to change your Ethernet IP settings)
  26. Cross-reference the excel spreadsheet to ensure you are entering the correct IP address for the displayed MAC address
  27. Enter the new IP address, should be 10.210.etc
  28. Enter the provided Subnet and Default gateway
  29. Before clicking save, confirm everything is entered correctly because the camera is about to reboot and have a new IP
  30. If you're ready, click Save.
  31. Repeat for all new cameras that are undergoing first time configuration

Once all cameras are configured, you may add them to the IVC Relay Server using the following how-to: IVC Camera Relay Server

If you have to bench test a camera that has an SDF IP address (10.210.ect) you do not need to connect to the SDF, you do need to change your Ethernet IP settings so the IP address is in the same range as the SDF IP address


Setup at the switch

  1. Disconnect laptop from WiFi
  2. Connect PoE Injector to a power source
    1. If the light doesn't turn on, choose another outlet (especially if you are in closet A8 where all but one outlet is disconnected)
  3. Plug ethernet cord connected to laptop into the LAN port on the PoE injector
  4. Unplug the ethernet cord from the port assigned to the camera you are configuring (the excel spreadsheet lists all of their labels in the Switch Labels column). The camera will now be disconnected from power and network
  5. Plug ethernet cord in so that one end is in the PoE port on the PoE injector and the other end is in the now empty port assigned to the camera you are configuring
  6. While the camera power cycles, open Network Connections on the laptop
  7. Right-click on the ethernet that is showing as connected and select Properties
  8. Double-click the Internet Protocol Version 4 (TCP/IPv4) connection
  9. With the Use the following IP address radio button selected,
    1. Manually set the IP address to an IP that is within range of the camera's IP
      1. i.e. If the camera's current SDF IP is 10.210.0.140, set the laptop to 10.210.0.139
    2. Set the Subnet mask to the same domain as the camera
      1. This should be 255.255.255.0
  10. Click Ok on both windows to return to Network Connections
  11. Wait for the selected Ethernet to say Connected, this can take some time, especially if the camera isn't done power cycling
  12. Open an internet browser
  13. Enter the IP address of the camera you are trying to connect to

If for some reason you do not know what camera you are connecting to, either use WireShark or enter IPs from the list of assigned IP addresses until one loads.

From here either start at step 13 from above or complete whatever configuration you are logging in to perform.

From your Laptop (currently this method isn't working reliably)

  1. First, find out who can connect to the SDF here: How to Get Access Through the SDF
  2. Connect to the SDF VPN
    1. If you are at a GM facility, choose INF SDF Warren
    2. If you are off-site, choose EXT SDF Warren
  3. Once Cisco confirms that you are connected you may proceed.  Connection can take up to a minute
  4. Open your network settings and select Proxy
  5. From the Proxy settings page, deselect the Use a proxy server slider.  It should slide to say Off
  6. Open an internet browser
  7. Using the Relay Server or the Excel spreadsheet for reference, enter the IP address of the camera you wish to connect to
  8. At the camera login page enter the default camera credentials

    1. Find the credentials Z:\Software Team\_LSIESystems.ini

    2. If neither works, email IVC Support with the camera model type and serial number
  9. You now have access to the camera directly 

Camera IP Address Unknown

The IP address for an unconfigured camera is on the camera box and on the underside of the device.  If you are using a camera that has been reset to factory defaults, the default IVC camera IP is 192.168.1.108

If the camera is configured, or not set to the default you must download, install, and use WireShark 

  1. Connect to the camera via Ethernet (see above for instructions)
  2. Launch WireShark
  3. Set the filter to arp and hit Enter
  4. Note to Chelsea: Screenshot this process
Remember: When accessed directly, the cameras have a different username and password than the IVC Relay Server.  Do not change the passwords.

What to do When we Lose Connection to a Camera

The best option is to send a message to Kai Wang that includes the MAC address, suspected IP address, and the port number for the camera.  He can either determine if

  1. it is on the wrong VLAN
    1. If the camera isn't on VLAN 752 please email James Currie and/or Manish Oza with the same information, letting them know the camera is on the wrong VLAN
  2. the port needs to be bounced*
    1. If the camera was on the wrong VLAN, it will need Kai to bounce the specific port the camera is attached to
