BTSI Lab Systems Integration : Quick Update

Created by Benjamin Fenner [X] on Oct 11, 2023


Things to keep in mind when updating container files. 

  • Version of the BRFM being used.
  • Version of the modules.
  • Weather the DUT is a dual or single manager. (Pretty much any DUT with more than 12 modules is dual)
  • If cheetah boxes are present. 


BEV3 12 modules or less Pack.

  • With manual coms open:
    • 2 - Get Cheetah device info
      • confirm that that there is one available.
    • 4 - Query manager info
      • The version should match the BRFM version you have selected.
    • 8 - Set Manager System Mode
      • enter 2, Commissioning Mode
    • 12 - Get Network Status
      • We are looking for all modules/nodes to be connected for each MAC address. Could take more than a few queries (reentering 12) before all connect.
      • Sometimes some never say "connected", but it's okay to move forward.
      • Other times if all do not connect and it's been about three minutes sense the first connection this usually means that the BRFM and modules are different versions. Inform the VE.
    • 18 - Send File to Device
      • You can update container files in any order (BMS, PMS, or EMS).
      • Do send the correct files look into the current version of the BRFM folder and copying the .bin file.
        • C:\Apps\ADT\iTest4.1\solution_common\EXECUTE\BRFM_Comm
      • If there is an error retry
    • 22 - Get Container Details
      • Confirm that the container files have been updated. In each of the folders with the .bin file there is a .h file with the hash to compare too.


BET modules or less Pack.

  • With manual coms open:
    • 2 - Get Cheetah device info
      • confirm that that there are two available.
    • 4 - Query manager info
      • The version should match the BRFM version you have selected and is a DUAL manager.
    • 8 - Set Manager System Mode
      • enter 2, Commissioning Mode
    • 12 - Get Network Status
      • We are looking for all modules/nodes to be connected for each MAC address. Could take more than a few queries (reentering 12) before all connect.
      • Sometimes some never say "connected", but it's okay to move forward.
      • Other times if all do not connect and it's been about three minutes sense the first connection this usually means that the BRFM and modules are different versions. Inform the VE.
    • 18 - Send File to Device
      • You can update container files in any order (BMS, PMS, or EMS) PMS and EMS need to be updated for both managers. The managers have both A and B manager container files.
        • The bt3a  files needs to be the first manager. 
      • Do send the correct files look into the current version of the BRFM folder and copying the .bin file.
        • C:\Apps\ADT\iTest4.1\solution_common\EXECUTE\BRFM_Comm
      • If there is an error retry
    • 22 - Get Container Details
      • Confirm that the container files have been updated. In each of the folders with the .bin file there is a .h file with the hash to compare too.

    • 8 - Set Manager System Mode

      • Sometimes some never say "connected", but it's okay to move forward.

    • 18 - Send File to Device
      • You can update container files in any order (BMS, PMS, or EMS) PMS and EMS need to be updated for both managers with the same container files. Module testing usually has one manager.
      • Do send the correct files look into the current version of the BRFM folder and copying the .bin file.
        • C:\Apps\ADT\iTest4.1\solution_common\EXECUTE\BRFM_Comm
      • If there is an error retry
