Overview¶

File synchronization allows you to automatically download solution files from LabCentral. Changes to these files can then be propagated to the station automatically via iTest's Auto-Patch feature. This feature is supported by the moduledefinition.xml file located in a station's $SYSTEMDIR or $SUPPORTDIR.

WARNING: The <icentral> node in the $SYSTEMDIR\moduledefinition.xml file is controlled from LabCentral via its Solution Patches feature. Local changes to this file will be overwritten. Local changes should be made to the $SUPPORTDIR\moduledefinition.xml file, as this file is not maintained by LabCentral.

 

Centralizing Files Between Test Systems¶

To centralize a file or a group of files between test systems, edit the iCentral section of the moduledefinition.xml file.

Example:

<icentral>
    <sync>
        <file>*.sc2</file>
        <localfolder>$SUPPORTDIR/Schedules</localfolder>
        <remotefolder>Schedules</remotefolder>
        <silent>FALSE</silent>
        <newfiles>TRUE</newfiles>
    </sync>
    <sync>
        <file>*.pkl</file>
        <localfolder>$SUPPORTDIR/Picklist</localfolder>
        <remotefolder>Picklists</remotefolder>
        <silent>FALSE</silent>
    </sync>
    <sync>
        <file>xducer.rng</file>
        <localfolder>$SYSTEMDIR</localfolder>
        <remotefolder>Global</remotefolder>
        <silent>TRUE </silent>
    </sync>
    <sync>
        <file>*.lol</file>
        <localfolder>$SUPPORTDIR/data</localfolder>
        <remotefolder>LogLists </remotefolder>
    </sync>
</icentral>

Synchronization Tags¶

The following tags are used to define how files are synchronized between multiple systems:

Tag Descriptions

Tag Description
<localfolder> The name of the local folder on the test system.
<remotefolder> The relative path of the remote folder on the LabCentral system.
<file> A file name filter describing one or more files in the remote folder. Wildcard characters are supported.
<newfiles> (Optional) When set to TRUE, any files in the remote folder that are not in the local folder will be treated as new files and downloaded. Otherwise, only files that match files in the local folder will be downloaded to the test station. By default, this value is set to FALSE.
<silent> When set to TRUE, the file is considered necessary and must be downloaded to the test station after triggering auto-patching. This will result in the file not being listed in the Select Patches dialog on the test station. If the AUTOPATCHMODE setting is set to AUTO, then all files are considered necessary and will be downloaded to the test station.
