Customizing Moduledefinition.xml File¶

To support file synchronization with LabCentral, the <icentral> node is added to the moduledefinition.xml file. It contains the details for the synchronized folder types. This node must be within the file's primary <Solution> node and should be the last listed node. In the example below, the system is configured to synchronize schedule (.sc2), picklist (.pkl), and log order list (.lol) files.

WARNING: Modification of the <icentral> node in the $SYSTEMDIR\ModuleDefinition.xml file is overseen from the LabCentral server. All local changes will be overwritten. Any $SUPPORTDIR\ModuleDefinition.xml files are not maintained by the LabCentral server, so local changes to these files will be left untouched.

 

<icentral>

<sync>
<file>*.sc2</file>
<localfolder>$SUPPORTDIR/Schedules</localfolder>
<remotefolder>Schedules</remotefolder>
<silent>FALSE</silent>
<newfiles>TRUE</newfiles>
</sync>

<sync>
<file>*.pkl</file>
<localfolder>$SUPPORTDIR/Picklists</localfolder>
<remotefolder>Picklists </remotefolder>
<silent>FALSE</silent>
</sync>

<sync>
<file>*.lol</file>
<localfolder>$SUPPORTDIR/data</localfolder>
<remotefolder>LogLists </remotefolder>
</sync>

</icentral>

Syntax¶

The sub-tags in the <icentral> node are used to define how files are synchronized between the two systems. You can use the following tags in your moduledefinition.xml:

<icentral> Tags

Tag Description
<sync> The tag that specifies a new group of files to include in the file synchronization.
<file> The name of the file or files to synchronize. You can also just specify the file type in this field. Wildcard characters are accepted.
<localfolder> The name of the folder on the iTest system that contains the files to synchronize (e.g., C:/ADT/iTest3.7/Solution.Demo/Picklist or $SUPPORTDIR/Picklist).
<remotefolder> The relative path of the remote folder on the LabCentral system (e.g., Picklists [the Picklists folder in LabCentral's Lab Files]).
<silent> The tag that determines whether the user has the option to download updated files from LabCentral or not. Set this setting to False to make all files an optional download. Set to True to be required downloads; if set to true, the file names will not appear in the Select Patch dialog.
<newfiles> This is an optional tag. The default value is False. If the tag is present and set to True, then any files in the remote folder that are not in the local folder are treated as new files and downloaded, matching files are also downloaded. Otherwise, only files that match files on the local folder are downloaded to the iTest system.
