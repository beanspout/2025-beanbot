Overview¶

iTest.exe is the primary application used to start iTest. It enables you to quickly open a specific display, layout, or interface, such as iTest Console or AutomationPanel. There are two INI options that are used to determine what opens when iTest.exe is launched: AutomationPanel and GUI.
 
iTest.exe Icon

 

INI Options¶

The following powertek.ini options specify iTest.exe functionality. Use them to customize which application or interface is opened when iTest.exe is launched.

AutomationPanel¶

This INI option determines which version of iTest is launched—either iTest Console or AutomationPanel. When this setting is true, AutomationPanel is launched; when it is false, iTest Console is launched. False is the default value for this setting.

Examples¶

AutomationPanel=true

AutomationPanel=false

GUI¶

This INI option is used when you do not want to open iTest Console or AutomationPanel—such as when using iTest on a touch screen device. When you launch iTest.exe, it will open the layout or display using DspDrv.exe (for iTest Console displays) and DspDrv_AP.exe (for AutomationPanel layouts). When this setting is not specified, iTest.exe uses the AutomationPanel setting to determine which application to launch.

Examples¶

DspDrv.exe¶

GUI=dspdrv.exe $SUPPORTDIR\EXECUTE\DISPLAY\Cas1_Combustion.dsp

DspDrv_AP.exe¶

GUI=dspdrv_AP.exe "$SUPPORTDIR\Layouts\2. Master Schedule.layout" /MainGui

Where:

/MainGui is required when you want to display a layout as the primary interface. When you close this layout, iTest shuts down. When you close layouts that do not have this argument, it will not shutdown iTest; instead, it will just close the layout.

NOTE: You must use quotation marks around the file path if the name of the file has spaces in it.

Read-only Mode¶

You can make iTest.exe launch in read-only mode by using the /readonly command line argument or the ReadOnly advanced powertek.ini setting.

Launching Scripts or Programs¶

iTest.exe will attempt to launch the following batch files in this order (if they exist) prior to launching any executables:

  1. $SYSTEMDIR\Execute\iTestPreload.bat
  2. $SUPPORTDIR\Execute\iTestPreload.bat

The batch files can be configured to run scripts or programs when iTest.exe is first executed. iTest.exe will set the following environment variables prior to launching these batch files:

  • DATADIR
  • EXECUTEDIR
  • SUPPORTDIR
  • SYSTEMDIR

These environment variables correspond to the similarly named iTest macros (e.g., $DATADIR, $SUPPORTDIR, etc...). They exist for the duration of this running instance of iTest.exe and are available to any child process launched by this instance of iTest.exe. For example, a batch file launched from AutomationPanel could also reference these variables.
