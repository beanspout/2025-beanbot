Overview¶

The ADT Data Entry display object allows you to configure a data entry field that accepts user-entered data during runtime on the iTest Console. The data that is input can then be used to assign a value into a channel. You can also use the ADT Data Entry control to ask users a question that allows them to input test data that is not measured by the data acquisition system.

Field Definitions¶

General Tab¶

General Tab

Option Description
Channel Name Click the Channel... button to open the Channel Selection dialog, which you can use to select the channel whose value will be updated by this control.
Input Type Select the type of data (Float, Integer, or String) entered in the ADTDataEntry control’s text field.
Prompt String Type the label that displays to the left of the text field.
Input Length Specify the number of characters that can be entered in the text field. The default is 10.
Input Precision Specify the number of decimal places (precision) for a number. For example, a value of 2 would limit the value to 2 decimal places (e.g., 4.32).
When this property is set to Default, the precision for the channel associated with the ADTDataEntry control is obtained from the channel's settings.
Data Destination Select the location in which to store the text entered in the text field. You can select DatabaseOnly or any .ini file. If an .ini file is selected, data is still sent to the database as well.
Control ID Enter a Control ID. If you have multiple Data Entry controls and will be programmatically changing them in any way (e.g., sending a mailslot to update the display), you will need a different Control ID for each. If this is the only Data Entry control on the display or you have multiple that you won’t be changing programmatically, use the ADTDataEntry default.
Show Border By default, this will shows the outer border for the control.
Permit Runtime Changes Deselecting this box prohibits users from making changes programmatically. It also makes this control read-only.

 

ADT Data Entry Object Properties¶

These properties are useful when issuing the following mailslot message:

\\.\mailslot\DisplayName>>$SETITEMPROPERTY ControlID PropertyName PropertyValue

The following are the names of the properties that can be changed by mailslot message to the display. These property names may or may not be the same as the property names shown on the display object property setup dialogs. Most of these property names are self-explanatory. The first column is the data type of the property.

The int or OLE_COLOR for color is the color RGB values. “BSTR” means the type of the property is string. “int”, “float”, “long”, “short”, “double” means the type of the property is number. “Boolean” means the property value should be 1 or 0.

Font property syntax is complicated.
<font name>,<bold>,,<italic>,<font size>,,<strikeout>,<underline>,
example: Arial,0,0,0,97500,0,0,0,400
