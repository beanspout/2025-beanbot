Overview¶

iTest supports map tables, which are also referred to as breakpoint or 2D tables. This table type can be used to linearly interpolate values in iTest. The following sections discuss map table format and the process for setting up map tables for automatic use in Test Manager. For more information on manually controlling map tables, see the Advanced Map Table Usage document.

Map Table Format¶

A map table is an X by Y grid of double precision values. The first row contains the breakpoint values for the X-axis, and the first column contains the breakpoint values for the Y-axis. This is a tab delimited file that uses the .2dt file extension.

Restrictions¶

  • Blank lines cannot be present between rows.
  • The use of channel names or string values within the map table is not supported.

Example¶

The first row identifies the X lookups, and the first column identifies the Y lookups. This example table is x size = 5, y size = 4.

Y/X 10 20 30 40 100
1 11 20 30 35 40
2 10 20 30 35 40
3 10 20 30 40 40
6 10 20 30 50 50

 

Adding the Maps Folder to Test Manager¶

To create and edit maps in Test Manager, the maps editor must be manually added to the moduledefinition.xml file. To add the editor to Test Manager, do the following:

  1. Navigate to your $SUPPORTDIR.
  2. Open the moduledefinition.xml file in a text editor.
  3. Add the following XML element to the XML System type tag.
    <folder>Maps</folder>
  4. Save the file.
  5. Open Test Manager. The Maps folder is visible in the tree bar as a sub-folder in the Test Development folder.

Creating New Maps¶

Creating new maps in Test Manager is quite simple. To create a new map, do the following:

  1. Right-click the Maps folder and select New Map.
     
    Add New Map

     
  2. In the Input Table Size dialog, enter the number of columns and the number of rows for the map table.
     
    Table Size

     
  3. Enter the X and Y lookup values in the first row and the first column. Then enter values in the remaining cells.
     
    Map Editor

     
  4. Click the save icon or select File > Save from the main menu to save the file.

Specifying Map Properties¶

Map properties allows you to automate map table handling. When the Enable checkbox is selected in the Map Properties dialog, iTest will automatically load the map table into memory at start-up and interpolate table values in the system's calculations at the rate of your choosing. The result channel stores the interpolated value based on the X and Y channel value lookups. In the case of an error, the result channel will be set to -99999.

To set table properties, do the following:

  1. Right-click in the table and select the Properties option.
     
    Properties

     
  2. In the Map Properties dialog, click the Enable checkbox to automatically use the map table during runtime at system scan rate.
     
    Map Properties Dialog

     
  3. Click the buttons to choose channels for the X, Y, and Result channels.
     
    Channel Selection Dialog

     
  4. Use the Channel Selection dialog to select a channel, and then click OK.
  5. Click the drop-down box next to the Rate (Hz) field to select the rate at which the map table is run.
     
    Rate

     
  6. Click OK when you are finished.

 

NOTE: Any status change to the Enabled checkbox or changes to the Map Table contents are only reflected after the next rebuild or restart of iTest.

 

WARNING: By default, iTest initializes channel values to 0 on startup. This will result in X and Y lookup channel values being initialized to a value of 0. This must be taken into account when designing your application. Channel values can be initialized to some other value by storing channel values in an INI file or using the persistent value setting for virtual output channels.
