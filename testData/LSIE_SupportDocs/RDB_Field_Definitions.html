RDB Field Definitions¶

The following is a list of configurable RDB fields:

Active Input Digital Input
PWM Input		 X - For a digital input, the Active Input setting is used to indicate whether an energized digital input will return a logic "1" or "0" to the iTest channel. If the Active Input checkbox is checked, then a logic "1" will be returned. If the setting is not checked, then a logic "0" will be returned. The channel shows the percentage of time it takes to energize the input. For more information, refer to the PWM I/O Support documentation.
Active Output Digital Output
PWM Output		 X - For a digital output, the Active Output setting is used to indicate whether a logic "1" or "0" from iTest will energize the digital output. If the Active Output checkbox is checked, then a logic "1" will energize it. If the setting is not checked, then a logic "0" will energize it. The channel shows the percentage of time it takes to energize the output. For more information, refer to the PWM I/O Support documentation.

Data Source Virtual Output
Virtual String 		- X The DataSource field is used to support channel updates from remote workstation PCs. The entry for DataSource is a hard-coded remote PC name or the name of a string channel that contains the remote PC name. For a channel defined this way, iTest on the local PC retrieves the channel value from the remote PC and updates the channel value on the local PC.
Example:
1. The flgShutdown channel is set to 1 by a master PC when all networked workstation PCs need to stop their tests.
2. The flgShutdown channel is defined on the master PC (with no DataSource) and on workstation PCs (with a DataSource of vsMasterPCName).
3. When the master PC sets flgShutdown to 1, all the network PCs see the change as well.
Debounce Digital Input X - The value screens for erratic data that can sometimes be collected from the digital input device. It is an integer value with units in seconds. If the change in state occurs for a period that is longer than the debounce time, the state change will be registered. This function can effect system performance if set too high.

Filter Time Constant Temperature

Persistent Value Virtual Output
Virtual String
Virtual Output Array 		X X Enables the automatic saving of channel values upon exit of iTest Console or Automation Panel. The persistent channel is saved to an .ini file, in the following order:
 
1. If the channel is in a module's database file, and the module .ini file is found, write to the module .ini file. The module .ini is assumed to be <ModuleID>.ini. The module database file is assumed to be named <ModuleID>.fil.
2. If the module .ini file is not found, and it is a modular solution, write to sys.ini if sys.ini is found.
3. If the module .ini and sys.ini files are not found, or the solution is not modular, write to the master.ini file.
 
This save is always performed on exit, so long as there has been successful communication between Data_Engine and the vxin_exec. It does not take into consideration if a test is running or even if communication is no longer occurring when the exit is performed.

User Defined1 - - X Used by iTest system code to indicate if the channel is available for use in set value tables, step tasks, and circuits. These channels could be used, for example, as a step parameter, a step operand, or both in a Step Condition (EOS). FlexEdit uses this to indicate if an Analog Input or Analog Output channel is to be listed in the AIM or AOM editors. This is tied to the <checkuserdefined1> FlexEdit XML tag.
 
Supported values are:
>0: Indicates that the channel is available in the AIM and AOM editors.
2: Indicates that the channel is available as a parameter.
4: Indicates that the channel is available as an operand.
 
iTest BTS systems only:
8: Indicates that the channel is available as an assignable circuit channel in the Circuit Editor.
16: Indicates the channel is a user defined channel for a circuit calculation or parameter.
32: Indicates that the channel is available for use when configuring the Voltage column for a schedule.
64: Indicates that the channel is available for use when configuring the Temperature column for a schedule.
128: Indicates that the channel is available for use when configuring the Current column for a schedule.
256: Indicates that the channel is available for use when configuring the Power column for a schedule.
512: Indicates that the channel can be used as a column for the Circuit Monitor.
1024: Indicates that the channel is available for use when configuring the Profile column for a schedule.
 
The values are combined to indicate that the channel can be used for more than one purpose. For example, a value of 6 indicates that the channel can be used as both a step parameter and a step operand.

Runtime Only RDB Fields¶

The following runtime RDB field's statistics are calculated by vxin_exec if a statistics channel is enabled in Test Manager's Channel Editor:

Runtime Only RDB Fields

Options Description
Maximum Specifies the maximum value of the channel during the window period set by iTest.
Mean Specifies the mean or average value of the channel during the window period set by iTest.
Minimum Specifies the minimum value of the channel during the window period set by iTest.
Slope Returns the slope (rate of change) value for the channel; it is calculated over the Statistical Window period.
Stable Set to 1 (true/stable) by iTest if the channel has met stability criteria for the duration of the window period; otherwise, this is 0 (false/unstable).
Std Deviation Returns the standard deviation value for a channel; it is calculated over the Statistical Window period.
