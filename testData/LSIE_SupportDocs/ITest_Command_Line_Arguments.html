Command line arguments can be used when launching some of the executables included with iTest.

Supported Executables¶

The following executables are supported:

  • AutomationPanel.exe
  • iTest.exe
  • iTestConsole.exe
  • TestMgr.exe
  • SolutionBuilder.exe

Arguments¶

/Solution¶

To launch a supported executable with a specific solution, use the following syntax:

<Executable> /Solution <SolutionDirectoryName>

NOTE: The solution must exist under the $SYSTEMDIR in order to use this switch. Reference the solution by its name or a fully-qualified path.

Example

iTest.exe /Solution Solution.Demo

/NoInitErrDlg¶

To suppress start-up error messages regarding the /Solution command, use the following syntax:

<Executable> /Solution <SolutionDirectoryName> /NoInitErrDlg

Example

iTest.exe /Solution Solution.Demo /NoInitErrDlg

NOTE: Other error messages that may appear during runtime are not blocked.

/skipselectsupport¶

To launch the solution specified in the SupportDir config.ini setting without opening the Select Support dialog, use the following syntax:

WARNING: The solution must exist in order to use this switch. 

iTest.exe /skipselectsupport
NOTE: This switch is not supported for SolutionBuilder.exe and TestMgr.exe.

Error Messages¶

The command's exit code is set to -1, 1, 2, or 3 depending on the error that caused the program to close. Error messages include, but are not limited to, the following:

Error Messages

Error Code Error Message Subject
-1 Another instance of SolutionBuilder is already editing the same solution.
1 SUPPORTDIR did not exist.
2 SUPPORTDIR was not a valid solution.
3 Another instance of iTest/AutomationPanel is running. Note: If another instance of Test Manager is running, an error dialog will not display.
