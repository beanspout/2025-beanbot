Overview¶

This document details the process of setting up watchdogs in cases where there is a loss of communication from either the vxin_exec or Data_Engine.

vxin_exec Watchdog¶

To set up a watchdog for the vxin_exec, refer to the iTestWD Setup documentation. This will enable the watchdog program, which will monitor the vxin_exec in case there is a loss of communication. When there is a loss of communication from the vxin_exec, the iTestWD.bat file is launched. For more information, refer to the iTestWD Operation documentation.

Data_Engine Watchdog¶

The vxin_exec monitors Data_Engine in case there is a loss of communication. When the vxin_exec detects a loss of connection after the time limit set in the DataEngineDisconnectTimeSecs powertek.ini setting has expired, it will run the VCL procedure specified in the DataEngineDisconnectProc powertek.ini setting. There will not be any visual indication in iTest that this procedure has been executed; however, the vxin_exec.log file is updated, noting the executed procedure. This provides a method to set connected devices into safe states rather than have them continue to run in an uncontrolled manner.

For more information regarding the DataEngineDisconnectTimeSecs and DataEngineDisconnectProc powertek.ini settings, refer to their entries in the iTest INI Options documentation.
