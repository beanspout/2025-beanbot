Overview¶

VCL Schedule functions are used to read and write schedule (.sc2) files programmatically.

Error Codes¶

Each VCL Schedule function returns a numeric code or a string when the operation is complete. This is usually captured using the SET VCL command. The channel will be populated with either a 0 (success) or some negative value (failure) if capturing the status using a numeric channel. If using a string channel to capture the function status, then it will be populated with either an empty string (success) or some descriptive text detailing the error. See the following table:

Return Values and Descriptions

Return Value Description
0 Table operation successful.
-1 Table operation failed.
-2 Table error, not loaded.
-3 Table error, function not supported.
-4 Page error, page not found.
-5 Page error, bad index value.
-6 Page error, invalid operation.
-7 Page error, keyword not found.
-8 Page error, data not cellwise.
-9 Function error, bad parameter value.
-10 Table error, file not found.
-11 Function error, bad parameter type.
-12 Table warning, exists in memory.
-13 Table error, insufficient memory.

 

Schedule Functions List¶

ColumnChannelGet¶

Sets a string channel to the name of the iTest channel associated with the given column on a schedule's matrix page.

Syntax

columnchannelget(<schedulename>, [pagename], <columnindex>, <resultchannel>)

Parameters

schedulename: Name of the schedule stored in memory. The <schedulename> parameter requires the .sc2 file extension and must match the schedule name originally used with the TableLoad VCL function.

pagename: Optional parameter. The name of the matrix page. If omitted, the function uses the page name from the TableAndScheduleDataPageName powertek.ini setting.

columnindex: Integer signifying the index of the column in the matrix page. This is a 1-based index.

resultchannel: String channel or local variable that is set with the name of the channel associated with the given column.

Examples

SET lclStatus columnchannelget("NewSchedule1.sc2", "Data", 1, vsColumnChanName)
SET lclStatus columnchannelget("NewSchedule1.sc2", 2, vsColumnChanName)

 

ColumnChannelSet¶

Changes a column's associated iTest channel on a schedule's matrix page. This change is made to the copy of the schedule stored in memory and will not modify the schedule (.sc2) file until the TableStore VCL function is called.

Syntax

columnchannelset(<schedulename>, [pagename], <columnindex>, <channelname>)

Parameters

schedulename: Name of the schedule stored in memory. The <schedulename> parameter requires the .sc2 file extension and must match the schedule name originally used with the TableLoad VCL function.

pagename: Optional parameter. The name of the matrix page. If omitted, the function uses the page name from the TableAndScheduleDataPageName powertek.ini setting.

columnindex: Integer signifying the index of the column in the matrix page. This is a 1-based index.

channelname: The name of the channel that is to be associated with the given column. This can be a string stored in a channel or local variable.

Examples

SET lclStatus columnchannelset("NewSchedule1.sc2", "Data", 1, vsColumnChanName)
SET lclStatus columnchannelset("NewSchedule1.sc2", 2, vsColumnChanName)

 

ColumnIndexGet¶

Gets the column index for the column in a schedule designated by the column title passed as a parameter to this function. The column title is the identifier displayed in the Schedule Viewer tool. This may differ from the name of the channel associated with the column.

Syntax

columnindexget(<schedulename>, [pagename], <columntitle>, <resultchannel>)

Parameters

schedulename: Name of the schedule stored in memory. The <schedulename> parameter requires the .sc2 file extension and must match the schedule name originally used with the TableLoad VCL function.

pagename: Optional parameter. The name of the matrix page. If omitted, the function uses the page name from the TableAndScheduleDataPageName powertek.ini setting.

columntitle: Title of the column to be searched for on the given matrix page.

resultchannel: Numeric channel or local variable that is set with the index of the column that matches the given title. Set to 0 if the given title does not match any columns listed on the given schedule's matrix page.

Examples

SET lclStatus columnindexget("NewSchedule1.sc2", "Data", "DynoSetpoint", voColumnIndex)
SET lclStatus columnindexget(vsScheduleName, vsColumnTitle, voColumnIndex)

TableClose¶

Removes a schedule from memory. After this function is run, the schedule will no longer be available until it is loaded back into memory using the TableLoad VCL function.

Syntax

tableclose(<schedulename>)

Parameters

schedulename: Name of the schedule stored in memory. The <schedulename> parameter requires the .sc2 file extension and must match the schedule name originally used with the TableLoad VCL function.

Example

SET lclStatus tableclose("NewSchedule1.sc2")

 

TableColumnMax¶

Sets an iTest channel to the maximum value stored in a schedule's numeric column.

Syntax

tablecolumnmax(<schedulename>, <pagename>, <columnindex>, <returnchannel>)

Parameters

schedulename: Name of the schedule stored in memory. The <schedulename> parameter requires the .sc2 file extension and must match the schedule name originally used with the TableLoad VCL function. Array channels can not be used for this parameter.

pagename: The name of the matrix page.

columnindex: Integer signifying the index of the column in the matrix page. This is a 1-based index.

resultchannel: Numeric channel or local variable that is set with the largest value found in the given channel. Array channels can not be used for this parameter.

Example

SET lclStatus tablecolumnmax("NewSchedule1.sc2", "Data", 1, voColumnMax)

TableLoad¶

Loads an existing schedule (.sc2) into memory. This function must be used prior to any other Schedule VCL function being used; otherwise, a -2 error will be returned by that function.

Syntax

tableload(<schedulename>)

Parameter

schedulename: This parameter requires the .sc2 file extension and does not support $macro substitution (e.g., $SUPPORTDIR). If no file path is provided, the file path from the TableFilePath powertek.ini setting is used (defaults to $SUPPORTDIR\SchedulesAndTables). If a fully qualified file path is provided, the schedule must always be referred to by the fully qualified path in subsequent function calls.
If the TableFilePath powertek.ini setting points to a non-$SUPPORTDIR location (e.g., $SYSTEMDIR or C:\Temp\), then table functions will fail under a 2PC system configuration.

Example

SET lclStatus tableload("NewSchedule1.sc2")

TableStore¶

Saves the given schedule in memory to disk. This overwrites the content of the existing schedule (.sc2) file.

This function does not support saving the schedule to any location other than the location originally referenced when the schedule was loaded into memory by the TableLoad VCL function.

Syntax

tablestore(<schedulename>)

Parameter

schedulename: Name of the schedule stored in memory. The <schedulename> parameter requires the .sc2 file extension and must match the schedule name originally used with the TableLoad VCL function.

Example

SET lclStatus tablestore("NewSchedule1.sc2")

 

TableValueGet¶

Sets a channel or local variable to the value of an entry on a given matrix or list page in a schedule.

Syntax

tablevalueget(<schedulename>, [pagename], <row, column>|<keyword>, <resultchannel>)

Parameters

schedulename: Name of the schedule stored in memory. The <schedulename> parameter requires the .sc2 file extension and must match the schedule name originally used with the TableLoad VCL function.

pagename: Optional parameter. The name of the page to be searched. If omitted, the function uses the page name from either the TableAndScheduleHeaderPageName or the TableAndScheduleDataPageName powertek.ini setting depending on whether the <keyword> or the <row>,<column> parameters were given.

row, column: Matrix pages only. The row and column to get the value from on the given matrix page.

keyword: List pages only. The keyword used to identify the value to get from the given list page.

resultchannel: iTest channel or local variable that is set with the found value. The found value is -1 if the channel associated with the column is a numeric type and the entry is a string value.

Examples

* referencing a matrix page:
SET lclStatus tablevalueget("NewSchedule1.sc2", "Data", 1, 2, voScheduleValue)
SET lclStatus tablevalueget("NewSchedule1.sc2", voRow, voColumn, voScheduleValue)

* referencing a list page:
SET lclStatus tablevalueget("NewSchedule1.sc2", "Header", "ScheduleStatusChannel", vsHeaderStr)
SET lclStatus tablevalueget("NewSchedule1.sc2", vsKeyword, vsHeaderStr)

 

TableValueSet¶

Sets the value of an entry on a given matrix or list page in a schedule. Setting both numeric and string values is supported.

Syntax

tablevalueset(<schedulename>, [pagename], <row, column>|<keyword>, <newentry>)

Parameters

schedulename: Name of the schedule stored in memory. The <schedulename> parameter requires the .sc2 file extension and must match the schedule name originally used with the TableLoad VCL function.

pagename: Optional parameter. The name of the page to be changed. If omitted, the function uses the page name from either the TableAndScheduleHeaderPageName or the TableAndScheduleDataPageName powertek.ini setting depending on whether or not the <keyword> or the <row>,<column> parameters were given.

row, column: Matrix pages only. The row and column to be changed on the given matrix page.

keyword: List pages only. The keyword used to identify the value to set on the given list page.

newentry: Value or string to set the entry to. When changing a list page, the new entry must be a string even if using numeric characters.

Examples

* referencing a matrix page:
SET lclStatus tablevalueset("NewSchedule1.sc2", "Data", 1, 8, "TransientLog1")
SET lclStatus tablevalueset("NewSchedule1.sc2", voRow, voColumn, voNewValue)

* referencing a list page:
SET lclStatus tablevalueset("NewSchedule1.sc2", "Header", "ScheduleStatusChannel", "vsNewStatusChannel")
SET lclStatus tablevalueset("NewSchedule1.sc2", vsKeyword, vsNewStringValue)
