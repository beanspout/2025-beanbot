Advanced Installations Guide¶

Overview¶

The following instructions describe the installation and upgrade process for PC-based iTest systems and will address most situations. For further information or assistance, please submit a support request using our customer portal at support.aanddtech.com.

Minimum System Requirements¶

The following configuration is recommended for an iTest Operator PC:

  • 2.1 GHz Core i7 PC
  • 230 Gigabyte Hard Drive
  • 16 Gigabytes of RAM
  • Windows 10 Pro version 1903 or higher
  • 1920x1080 resolution display is recommended; however, this is application dependent.
  • Discrete GPU with dual monitor support and 1 GB of display memory. Onboard GPUs (like Intel GMA) are NOT supported. Systems that have both discrete and onboard GPUs must be configured to use the discrete GPU exclusively. A&D recommends AMD Radeon 5450 class GPU or higher.
  • Ethernet adapters (minimum of 2 recommended)
    • Ethernet RJ-45 – Building Network
    • Ethernet RJ-45 – iConnect distributed I/O Network
  • USB 2.0 ports (4 recommended)
  • UPS (1500 VA recommended)

For the iTest real-time controller, the following configuration is recommended:

  • 3.3 GHz Core 2 Duo CPU
  • 230 Gigabyte Hard Drive
  • 4 Gigabytes of RAM
  • INtime RTOS
  • 20X DVD +/- RW drive
  • Ethernet adapters (minimum of 2 recommended)
    • Ethernet RJ-45 – iTest Operator PC
    • Ethernet RJ-45 – Test Cell Devices

Additional Windows Requirements:

  • You must be logged in as an Administrator to install iTest.
  • Ensure non-administrator users have full-control permissions for the iTest installation folder.
  • All users of Windows using version 7 or above must first modify the settings of the User Account Control (UAC) before starting installation. The UAC is a Windows service that notifies you before changes are made to your computer that requires administrator-level permission. The default UAC setting notifies you when programs try to make changes to your computer. For iTest, however, this setting needs to be changed to Never notify because iTest relies on being able to modify the registry to register its latest version of the display OCXs. Failure to disable this Windows feature can cause iTest to appear to hang or the display to not appear correctly.

Quick Install Procedure¶

Installation on a Real-Time PC¶

Two PC systems only:

NOTE: Only the GUI PC needs to be updated on upgrades after iTest version 3.2.

  1. Stop all iTest services on the real-Time PC.
  2. Install iTest using the dongle.
  3. Confirm that the newly created C:\ADT\iTestRT\ directory is shared in Windows.

Installation on a Windows GUI PC¶

NOTE: The list below is intended as a quick overview. If more detail is required, please refer to the remainder of this document.
  1. Make backups.
  2. Install iTest using the dongle.
  3. Install the iTest PatchCumulative files (download the latest from the A&D customer portal at https://support.aanddtech.com/home).
  4. Copy the current installed solution to the iTest directory that was created during this iTest installation.
  5. Modify config.ini with the previous system's settings if necessary (e.g., if hard-coded paths reference files in a previous version). Contact A&D if you're unsure about how to proceed with this step.
  6. For two-PC (real-time) systems, re-map the GUI PC's i: drive to C:\ADT\iTestRT\ on the real-time PC.
  7. For two-PC (real-time) systems, create the HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\LanmanWorkStation\Parameters\DirectoryCacheLifetime registry setting as a DWORD (32 bit) type and set the value to 0.

Backups¶

As general good practice, make backups of the entire directory of previous iTest versions (including subdirectories) before installing iTest to the GUI PC.

To save a backup of the iTest folder, do the following:

  1. Open Windows Explorer and navigate to the $SYSTEMDIR.
  2. Copy the iTestx.x folder.
  3. Paste the folder into a different location.
  4. Right-click the copied folder and select Rename.
  5. Type a new name for the folder that will easily distinguish it as a backup (e.g., C:\ADT_backup\iTestx.x_Backup).

Real-Time PC Installation¶

To install iTest on a real-time PC, do the following:

  1. Stop all iTest services on the real-Time PC.
  2. Create a backup of the existing directory.
  3. Locate the iTestx.xSetup.exe setup executable, where x.x is the product version number.
  4. Right-click iTestx.xSetup.exe and select Run as Administrator.
  5. In the User Account Control dialog, click Yes to allow changes to your device.
  6. Read the license agreement terms and conditions. Click the I accept the conditions option, and then click Next.

  7. Select the System Type "iTest Services for INtime (2nd PC)".

  8. Enter a name for the installation folder or click Next to accept the C:\ADT\iTestx.x\. default location.

  9. Enter your User Name and Organization. Leave the Install Key field empty; an installation key isn't necessary for iTest Services for InTime (2nd PC).

SystemConfig¶

This utility is first run after a new installation of most iTest installs. It can then be run manually at any time to change the global settings for the PC. SystemConfig is available in the Windows Start Menu.

SystemConfig

SystemConfig contains an editor for changing the following system settings. The majority of these settings are written to the Config.ini file which is located in the directory where iTest is installed. You can use SystemConfig to change the following settings:

SystemConfig Settings

Setting Description
iTest GUI Language Use this combo box to change the preferred language for iTest.
Enable/Disable LabAgent on this machine Enable/disable LabAgent. This setting is changed in the registry. Note: Your anti-virus may prohibit SystemConfig.exe from making this change. If it does, you will need to add an exception to your anti-virus for allowing SytemConfig.exe to change the start-up entries in your registry. LabCentral will not be able to communicate with your system if LabAgent is disabled.
LabAgent Port Number By default, this is 80. It can be changed in this field.
Automated Patching Mode (and Patch folder) Use these settings to set up a central location for patching iTest binaries (.exe, .dll, .tdaddin, and .ocx). Typically, this would be a LabCentral share name, but it can be any shared drive on your network. This feature allows you to put patches in one location on the network and patch all iTest systems.

This next group of settings should only be changed if you do not have a LabCentral system. If you have LabCentral, changing these settings will overwrite your LabCentral settings and could possibly break your connection to LabCentral.

SystemConfig Settings

Setting Description
iTest Module file location This gives you a network share to import and export modules to if you do not have LabCentral.
Enable iTest Security Enables the login prompt for iTest Console and Test Manager.
Enable Windows Domain Logins You can have iTest send the username and password to the Windows domain server to have the password verified. When using this feature, you still configure groups in the iTest security editor. However, when the user logs in with their domain username and password, the software checks to see what groups they belong to in the domain. If any of these groups match the iTest-configured groups, those groups define their security level.

Silent iTest Installation¶

Installation Pre-Checks¶

Prior to executing the silent installation, verify whether iTest is currently installed. This can be done by determining if the uninstall registry key, used by Windows to add and remove programs, exists on your system.

Example:¶

iTest registry key on a 64-bit Windows system:

HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft\Windows\CurrentVersion\Uninstall\{XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX}}_is1

iTest registry key on a 32-bit Windows system:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall\{XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX}}_is1

Perform the Silent Install¶

The iTest setup executable can be run in silent mode using the following command-line:

iTest38Setup.exe /VERYSILENT /SUPPRESSMSGBOXES

An install.ini file must be located in the same folder location in which iTest38Setup.exe is located, where the contents of the install.ini file include:

[Install]
SerialNumber=xxxx-xxxx-xxxx-xxxx

Where xxxx-xxxx-xxxx-xxxx is replaced with the actual iTest installation key (serial number).

You may additionally add a command-line parameter to support the deviation from the default directory:

iTest38Setup.exe /VERYSILENT /SUPPRESSMSGBOXES /DIR=x:\dirName

Enable LabAgent¶

Enabling LabAgent is done by running the registry key.

Example:

iTest registry key on a 64-bit Windows system:

HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft\Windows\CurrentVersion\Run\iView.exe

Example:

iTest registry key on a 32-bit Windows system:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run\iView.exe

Additional Manual Configuration¶

The following additional actions need to be taken upon completing any of the GUI PC iTest installations. These instructions do not need to be performed if you have just finished installing iTest on a two-PC (real-time) system.

  1. Transfer working solution from earlier version of iTest (this step is only necessary if you choose to import config.ini settings from an existing version of iTest):
    • Copy C:\ADT\iTestx.x\<your solution name> (e.g., Solution.Demo) folder into $SYSTEMDIR (default: C:\ADT\iTestx.x)
    • Copy C:\ADT\iTestx.x\config.ini into the $SYSTEMDIR (default: C:\ADT\iTestx.x).
  2. Select the appropriate file and rename it to data.def. There are multiple C:\ADT\iTestx.x\Installation\data.def files; contact A&D if it is not clear which file to select based on the list below:
    • data.def – standard, default data.def file.
    • data.def.1065 – required for use with the Calibrate 1065 feature. This must be used if customer is CFR 40 Part 1065 compliant.
    • data.def.vclref – required for use with the vclref feature. This feature is part of the dynamic customer name feature.
  3. Update the config.ini file to use the new iTest x.x paths:
    • Open the config.ini in the C:\ADT\iTestx.x folder using a text editor like Notepad. Change references of previous versions of iTest to your newly installed version of iTest.
      Example:
      Original: SystemDir=C:\ADT\iTest3.4\ 
Updated: SystemDir=C:\ADT\iTest3.8\
  4. Restart the computer.

You can now begin using the iTest software.

NOTE: If you are using drivers, insert your iTest dongle into a USB port before executing iTest. The iTest binaries (e.g., iTestConsole.exe and TestMgr.exe) will check for the presence of a dongle for certain drivers when they run. Refer to the Driver Licensing and Validation Tool section below for more information.

Driver Licensing and Validation Tool¶

When you install iTest, all available drivers are also installed. In order to use the drivers, you must have a valid security key enabled. The DongleUtil.exe utility shipped with your iTest software allows you to see which drivers you have valid access to. In addition, it also allows you to purchase additional security keys for drivers you need or will need in the future, as the test cell needs may expand. To see which drivers you have immediate access to, perform the following steps:

  1. Navigate to the $EXECUTEDIR and double-click DongleUtil.exe to launch the DongleUtil application.
  2. Input the dongle you received with the iTest software into the USB port on the computer. Wait a few seconds for the computer to recognize the hardware. Once it's recognized, a dialog populates a list of drivers that you have valid security keys for.
     
    DongleUtil

Upgrading an Existing Version¶

Supported users will be able to access an FTP site which may contain product fixes for their version of iTest. When these product fixes are available, they can be installed and their existing version of the application will be upgraded. In addition, the product fix installations are standalone patches, which do not require a full reinstallation of iTest.

Please refer to your Software Maintenance Agreement for more details on licensed software updates.

Uninstalling an Existing Version¶

If you are uninstalling a version that you have already done a considerable amount of work on, save a backup copy first. Refer to the Backups section above.

To uninstall an existing version, complete the following procedure:

  1. Navigate to your Control Panel by clicking Start > Control Panel.
  2. From the Control Panel, click the Programs and Features option.
  3. At this point, a list will be populated showing all of the applications/services currently installed on your computer. This may take a few seconds as some have more installed than others.
  4. Scroll down and highlight the iTest application from the list.
  5. Click the Uninstall button. The system will then ask you to confirm the removal of iTest, which is initiated by clicking Yes.
     
    NOTE: You may get a dialog box informing you that there are shared files that may be affected by removing this program. It’s always a good idea to click No to All just in case the files in question are referenced in another module of a program suite.

     
    Uninstall

     
  6. The system will then remove all associated program files from your machine.

Side-by-Side Installation Notes¶

When installing a new version of iTest, it is possible to use non-standard settings to retain your iTest installation on the RT PC.

On the GUI PC:

  • Drive letter. You may use a drive letter other than I:, but if you do, then you must specify it in the config.ini with the MappedRemoteDrive=X: setting (where X:_ can be any letter not in use).
  • Real-time PC name. If the RT PC name is not "iTestRT" or if the installed iTest folder on the RT PC is not iTestRT, you must specify it in the config.ini with the following setting: 
    RemoteShareName=\\machine_name\share_name

For example, RemoteShareName=\\iTestPC\iTest3.x.

To switch between versions after installation, you must first perform the following:

  • GUI PC: Run registerocx.bat in the iTestX.X\Execute folder you want to start using.
  • RT PC: Stop the iTestListen.rta process in INtime and start the iTestListen.rta process from the iTest folder you want to run from (e.g., iTestRT or iTestx.x).
  • RT PC: Stop the iTestBackground.exe process in Windows and start the iTestBackground.exe process from the iTest folder you want to run from (e.g., iTestRT or iTestx.x).
