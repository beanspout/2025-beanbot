Purpose¶

A utility that opens a Windows dialog entitled FileDialog. This dialog is used to browse for and select a file. The selected file’s name and file-path are then populated to designated virtual string channels in the RDB.

Command Line¶

Command Line 1: (string1 represents a file)

FileDialog.exe <string1>;<channel1>;<channel2> [-noext,/noext] [-ontop,/ontop]

Command Line 2: (string1 represents a folder)

FileDialog.exe <string1>;<string2>;<channel1>;<channel2> [-noext,/noext] [-ontop,/ontop]

Parameters

  • <string1>: Literal string, fully qualified file-path. If it includes a filename, then the text following the last backslash is populated to the File name field and acts as a filter on the displayed folder contents.
  • <string2>: Literal string, filename complete with extension. This literal string populates the File name field and acts as a filter on the displayed folder contents.
  • <channel1>: Virtual string channel name. The channel is populated with the selected file’s name if the Open button is clicked in the FileDialog dialog.
  • <channel2>: Virtual string channel name. The channel is populated with the selected file’s file-path, including the filename, if the Open button is clicked in the FileDialog dialog.
  • [-noext,/noext]: (Optional argument). If ‘-noext’ or ‘/noext’ is added to the end of the command line entry, then the file name stored in <channel1> will not include the file extension.
  • [-ontop,/ontop]: (Optional argument). If ‘-ontop’ or ‘/ontop’ is added to the end of the command line entry, then the FileDialog window will always be "on top."

Button Events:

  • [OPEN] = <channel1> and <channel2> are populated.
  • [CANCEL] = <channel1> and <channel2> are populated with the FileDialogCancelString powertek.ini setting value or Cancel if the setting isn’t present.

Example Command Line entry:

FileDialog.exe C:\ADT\iTest3.2\iTest.Demo\Schedules;*.sc2;vsStoreFileName;vsStoreFilePath –noext -ontop

FileSelect.exe Overview¶

Purpose¶

A utility that opens a Windows dialog entitled FileSelect. This dialog is used to browse for and select a file. The selected file’s name and file-path are then populated to designated virtual string channels in the RDB. An iTest procedure is run depending on which button is used to close the FileSelect dialog.

Command Line¶

FileSelect.exe <channel1>,<string1>,<channel2>,<channel3>,<string2>,<string3>,[-f or nofilename][ -e or noext]

Parameters:

  • <channel1>: Virtual string channel name populated with the folder location to be opened. Populates the Look in field in the FileSelect dialog.
  • <string1>: Literal string, filename filter including extension. This is populated to the File name and Files of type fields and acts as a filter on the displayed folder contents.
  • <channel2>: Virtual string channel name. The channel is populated with the selected file’s file-path, including the filename, if the Open button is clicked in the FileSelect dialog.
  • <channel3>: Virtual string channel name. The channel is populated with the selected file’s name if the Open button is clicked in the FileSelect dialog.
  • <string2>: Literal string. This is the name of a procedure to run if the Open button is clicked in the FileSelect dialog. The procedure is initiated using a SEQUENCE START mailslot message to data_engine. This argument can be left empty and a procedure will not be run.
  • <string3>: Literal string. This is the name of a procedure to run if FileSelect dialog is exited without clicking the Open button (i.e., Cancel or windows close). The procedure is initiated using a SEQUENCE START mailslot message to data_engine. This argument can be left empty and a procedure will not be run.
  • [-f,nofilename]: (Optional argument). If ‘-f’ or ‘nofilename’ is added to the end of the command line entry, then the file path stored in <channel2> will not include the file name.
  • [-e,noext]: (Optional argument). If ‘-e’ or ‘noext’ is added to the end of the command line entry, then the filename stored in <channel3> will not include the file extension.

Button Events:

  • [OPEN] = <channel2> and <channel3> are populated. The procedure designated in <string2> is run.
  • [CANCEL] = The procedure designated in <string3> is run.

Example Command Line entries:

FileSelect.exe vsDirectory,*.ini,vsStoreFilePath,vsStoreFileName,procOKButton,procCancelButton,-e -f

FileSelect.exe vsDirectory,*.ini,vsStoreFilePath,vsStoreFileName,,,
