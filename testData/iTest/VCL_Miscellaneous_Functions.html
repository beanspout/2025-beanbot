Overview¶

The following document lists miscellaneous VCL functions.

AliasToChanVal¶

Returns the value stored in a numeric channel. The numeric channel is referenced by a string stored in a local variable or a string channel. The stored string can be a channel name or an alias*. Invalid references will return -999.

Syntax

AliasToChanVal(<Channel>, <CircuitID | "">)

Parameters

<Channel>: A local variable or string channel containing the channel name or alias* of a numeric channel. The stored numeric channel must be stored by its channel name and not by a customer or module name.

<CircuitID | "">: In a battery solution (BTS), this parameter would be the CircuitID; otherwise, the parameter is an empty string.

* To ensure a successful return, select an Alias that is unique to a single channel. If you reference an Alias used by multiple channels, the returned value will be selected at random.

Example

SET lclValue AliasToChanVal(vsStringPlaceholder, "")

* Given vsStringPlaceHolder contains the alias for the numeric channel (voDynoSpt).
* Returns the value stored in the voDynoSpt channel.

bitclear¶

Sets a bit, numbered 1-32, of a channel to 0. <bit-number> is a numeric constant, local variable or channel name containing an integer value of 1-32. The change is made to the specified bit of the channel defined in the SET command.

Syntax

SET <ReturnValue> bitclear(<bit-number>)

Example

SET numSysBitChannel bitclear(5)

* Where numSysBitChannel previously had a bitset performed on bit 5 and on bit 1.
* In this example, bit 5 of numSysBitChannel would be set to 0, and the value of numSysBitChannel would be changed from 16 to 1.

 

bitset¶

Sets a bit, numbered 1-32, to 1. <bit-number> is a numeric constant, local variable or channel name containing an integer value of 1-32. The change is made to the specified bit of the channel defined in the SET command.

Syntax

SET <ReturnValue> bitset(<bit-number>)

Example

SET numSysBitChannel bitset(5)

* Where numSysBitChannel had no bits set yet, so numSysBitChannel value is 0.
* In this example, bit 5 of numSysBitChannel would be set to 1, and the value of the numSysBitChannel channel would be 16. If a bitset was also done on bit 1, numSysBitChannel would be 17.

 

bittest¶

Returns the bit value of a specified bit, numbered 1-32, in a numeric channel.

Syntax

SET <ReturnValue> bittest(<channel>,<bit-number>)

Example

SET numSysBitValue bittest(numSysBitChannel 5)

* Where numSysBitChannel previously had a bitset performed on bit 5.
* In this example, numSysBitValue would be set to 1.

 

boxcar¶

Returns the average value for an iTest channel over the last time duration (in seconds).
The second parameter must be a numeric constant.

Syntax

SET <ReturnValue> boxcar(<channel>,<seconds>)

Example

SET SpeedAvg boxcar(Speed, 2)

*In this example, the SpeedAvg channel value is set to the average value of the Speed channel over the last two seconds.

 

callingprocedureget¶

Sets a string channel or local string variable to the name of the parent (calling) procedure or “autotest” if it has no parent. This function returns 0 if the function completes successfully. A -1 is returned upon failure. A -4 is returned if the function returns an empty string.

Syntax

SET <ReturnValue> callingprocedureget(<channel>)

Parameter

channel: Set to the name of the calling procedure.

NOTE: This function will return an empty string if the procedure is "spawned" instead of "executed". Additional examples of this occurring would be if the procedure was run as a limit action or started using the debugger in Test Manager.

Example 1

* Given Procedure1 contains:
+0.0 EXECUTE Procedure2

* Given Procedure2 contains:
+0.0 SET voStatus callingprocedureget(vsParent)

* In this example, vsParent would be set to "Procedure1" after Procedure1 is run.

Example 2

* Given that Procedure1 contains:
+0.0 SPAWN Procedure2

* Given Procedure2 contains:
+0.0 SET voStatus callingprocedureget(vsParent)

* In this example, vsParent would be set to "" after Procedure1 is run.

 

channelnameget¶

Sets a string channel or local string variable, passed as the second parameter, to the name of the channel or procedure argument passed as the first parameter to the function. The procedure argument must be passed by ChanName. The returned value is based on the channel type of the receiving iTest channel.

Syntax

SET <ReturnValue> channelnameget(<channel1>,<channel2>)

Parameters

channel1: A channel name or a declared procedure argument. The argument must be passed by ChanName.

channel2: The name of a string channel or a local string variable.

Example

* Given that strChannel is a virtual string channel.

Example 1:
+0.0 SET strStatus channelnameget(CLOCK_TIME,strChannel)

* Given that strStatus is a virtual string channel.
* In this example, the function would set strStatus to either "" (if successful) or an English error message on error.

Example 2:
SET voErr channelnameget(CLOCK_TIME,strChannel)

* Given that voErr is a virtual output channel.
* In this example, the function would set voErr to either 0 (if successful) or a negative number on error.

Example 3:
*Parent Procedure:
EXECUTE ChildProcedure(CLOCK_TIME)

* Child Procedure:
$ARGUMENT arg1,Number,ChanName
SET voErr channelnameget(arg1, strChannel)

* In all examples, the function would set strChannel to "CLOCK_TIME" if the channel exists in the solution.

Limitations
  • If an argument has been passed as the first parameter, then the referenced channel must be of a numeric type like a virtual output. String channels are not supported.
     

dvdt¶

Returns the rate of change value for an iTest channel over the last time duration (in seconds).
The second parameter must be a numeric constant.

Syntax

SET <ReturnValue> dvdt(<channel>,<seconds>)

Example

SET SpeedDvdt dvdt(Speed, 2)

*In this example, the SpeedDvdt channel value is set to the rate of change value of the Speed channel over the last two seconds.

 

evalboolexpr¶

Evaluates an expression, represented by a string, and returns the result as a numeric constant; i.e., 0 (false), 1 (true), or -1 (error). Only a simple “operand operator operand” format is supported, so parentheses cannot be used to build complex expressions and the expression is evaluated strictly left to right. Supported operators are <, >, <=, >=, | and &.

NOTE: Numeric value comparisons are only to the first six (6) significant digits. For example, 1235740 1235749 and 3.96000 3.960009.

Syntax

SET <ReturnValue> evalboolexpr(<expression>)

Example

SET Cntr1 4
SET Cntr2 3
SET vsTemp "Cntr1 > Cntr2" 
SET Cntr3 evalboolexpr(vsTemp)

* In this example, the function would set Cntr3 to 1.

 

evaleos¶

Used to evaluate a Step Condition. The operands within the Step Condition can be either a channel name or an alias name. It can be an alias name only in a battery solution. In this case, it will be converted to a channel name using the circuit ID provided as the second argument of the function. This function returns either 1 (True), 0 (False), or -1 (Error).

NOTE: Numeric value comparisons are only to the first six (6) significant digits. For example, 1235740 1235749 and 3.96000 3.960009.

When used in a BTS environment, you must create a series of virtual output channels called num<CID>Circuit (<CID> = circuit ID, e.g., Cct01) There must be one for each circuit. These channels need to be set to the numeric value representing that circuit (e.g., for circuit 1, the numCct01Circuit channel must exist in the solution with a channel value of 1).

Syntax 

SET <ReturnValue> evaleos(<eos>,<circuitID>|"")

In a non-battery solution, the <circuitID> parameter is an empty string.

Example (non-battery):

SET voChan1 evalEOS("StepCondition1","")

 

evalfunc¶

Executes the specified procedure and returns its value.

Syntax 

SET <ReturnValue> evalfunc(<string|channel>)

Parameters

string | channel: Either a literal string surrounded by double quotes that matches the name of a procedure or a virtual string channel that contains the name of a procedure. The reference to the procedure can also include any arguments.

NOTE: To include literal strings as part of an argument, the string must be surrounded by double-quotes that have been escaped using tilde characters.

Example

Example 1:

SET voChannel1 evalfunc("NewProcedure1")

* given that the NewProcedure1 procedure returns 99.
* voChannel1 is set to 99.


Example 2:

SET voChannel1 evalfunc("ReturnNumberArg1(12, ~"abc~")")

* given that the ReturnNumberArg1 procedure returns its first argument.
* voChannel1 is set to 12.

Example 3:

VCLSTR lclString
SET lclString "ReturnNumberArg2(~"abc~", 12, 13)" 
SET voChannel1 evalfunc(lclString)

* given that the ReturnNumberArg2 procedure returns its second argument.
* voChannel1 is set to 12.
NOTE: EvalFunc() only accepts returned numeric values; it does not accept returned strings.

EvalSimpleExpr¶

Evaluates a simple string expression with two operands. The operands can be either channel name or an alias name. It can be an alias name only in a battery solution. In this case, it will be converted to a channel name using the circuit ID provided as the second argument of the function. This function returns a numeric value. Only a simple “operand operator operand” format is supported, so parentheses can not be used to build complex expressions. Supported operators are +, -, *, or /.

Syntax

SET <ReturnValue> EvalSimpleExpr(<Expression>,  <CircuitID>|"")

In a non-battery solution, the <CircuitID> parameter is an empty string.

Example

SET voResult EvalSimpleExpr("Voltage + 500", "")
SET voResult EvalSimpleExpr("Voltage + 500", "Cct1")

* Where "Voltage" is an alias or a channel name. For a battery solution with circuit "Cct1", the channel...
* ...name might be "eCct1Voltage".

 

fifo¶

Checks the status of the 'First In First Out' (FIFO) buffer and returns the value of the specified channel at a certain time in the past. The <time> parameter must be a numeric constant, in seconds, and can not be a reference to a channel or local variable.

A FIFO buffer is created for each channel referenced within a FIFO function in any existing iTest procedure (*.tsc files) in the solution. Only one buffer will be created for each channel, and its size is based on the provided <time> parameter and the system rate. Any subsequent FIFO commands for a channel will compare the <time> parameter versus the time encompassed by the existing FIFO buffer and increases the FIFO buffer size if the <time> parameter is larger. The FIFO buffer stores iTest channel values as single precision floating point numbers at the system rate, so a 10 second time parameter with a 100Hz system rate requires 4000 bytes of memory as iTest stores 1000 data points in the FIFO buffer for that channel. Each FIFO buffer is limited to no more than 100,000 data points.

WARNING: Multiple large FIFO buffers can degrade system performance as memory is allocated for these buffers.

Syntax

SET <ReturnValue> fifo(<channel name>,<time>)

Example

SET spdSysFifo5 fifo(spdSys,5)

* Where spdSys was 2875 5 seconds ago and is currently 3005.
* In this example, spdSysFifo5 is set to 2875.

 

GetProperty¶

Allows the operator to retrieve a single RDB property of an iTest channel from memory. The return value for the function is either a string or a numeric value depending on the RDB field specified.

The primary use case for this function is to retrieve the last average log value for a given channel instead of using FileServices. The GetProperty() VCL function's performance is real-time versus the high latency file I/O operations associated with FileServices.

Syntax

SET <ReturnValue> GetProperty(<channel name>,<field name>)

Parameters

channel name: An iTest channel or local variable name with no double quotes around it.

field name: A string constant (not an iTest channel or local variable) with double quotes around it.
The following values are supported for the field name:

"LastLoggedValue"
"Low EU"
"High EU"
"Low Rail"
"High Rail"
"Limit Status"
"Limits Active"
"Low Limit"
"Low Limit Action"
"Low Limit Ignore Time"
"Low Warning"
"Low Warning Action"
"High Warning"
"High Warning Action"
"High Limit"
"High Limit Action"
"High Limit Ignore Time"

Example

* to get the Low EU field for virtual output voTest1 the call would be:
+0.0 SET voRetVal GetProperty(voTest1, "Low EU")

 

lookup¶

Returns the non-linear value from a lookup table or its linearly interpolated value. The returned value is based on the value of the lookup channel for the table. For more information regarding lookup tables, refer to the Lookup Tables document.

Linear interpolation is used to compute the value of the return if applicable. Linear interpolation applies if the value of the lookup channel falls between two points in the table. If the LookupTableExtrapolate=FALSE powertek.in setting is present or missing, then RMISS is returned if the given value is outside of the range of the table. If LookupTableExtrapolate=TRUE is present, then linear interpolation is also applied if the value falls outside of the table's range. In this case, either the first or last two points in the table are used to interpolate the value depending on which table boundary was exceeded.

Syntax

SET <ReturnValue> lookup(<tablename>)

Parameters

channel: The numeric channel or local variable that stores the return value.

tablename: The name of a table created in Test Manager located within the Calculations/Lookup Tables tree node. Only a literal string is accepted for this parameter.

Example

SET valSysChannel lookup(VoltsToRPM)

* Where valSysChannel is a virtual output channel.
* Where VoltsToRPM is a lookup table referencing a 0v to 10v analog input channel in which 0v has a return value of 0 rpm and 10v has a 
return value of 5000 rpm.

In this example, the function returns an interpolated RPM value from the lookup table based on the current voltage value of the lookup channel.

 

lookupary¶

Returns the linearly interpolated value from a lookup table. The returned value is based on an index of the virtual output array channel that is being used as the lookup channel for the table. For more information regarding lookup tables, refer to the Lookup Tables document.

Syntax

SET <ReturnValue> lookupary(<tablename>,<arrayindex>)

Parameters

channel: The numeric channel or local variable that stores the return value.

tablename: The name of a table created in Test Manager located within the Calculations/Lookup Tables tree node.

arrayindex: The index of the virtual output array channel used to find the lookup value in the lookup table.

Example

SET tempReturnValue lookupary(CelsiusToFahrenheit, numIndexArray)

* Where tempReturnValue is a virtual output channel. 
* Where CelsiusToFahrenheit is a lookup table referencing a  0 °C  to  100 °C temperature channel within which 0 °C  has a return value of 32 
°F and 100 °C has a return value of 212 °F.
* Where numIndexArray is the index of the virtual output array channel referenced in the lookup table.

In this example, the function returns an interpolated Fahrenheit value from the lookup table based on the temperature value stored in an 
element of the virtual output array channel referenced in the table. The element being specified by the numIndexArray parameter.

 

lowpassfilter¶

Returns a filtered value based on the input channel and filtering coefficient. The filtering coefficient must be a constant value, a numeric channel, or a numeric array channel with a specified index.

The filter equation is as follows: OutputChannel[t] = fc*InputChannel[t] + (1 - fc)*OutputChannel[t-1]

In this equation, fc represents the filtering coefficient (0 > fc <= 1). The higher the value of fc, the more dynamic the output.

Syntax

SET <output channel> lowpassfilter <input channel, filtering coefficient>

Example

SET voDmo1Feedback lowPassFilter(voDmo1Square, 0.01)

msgwait¶

Sends a mailslot message, periodically compares the value of the channel passed to the function against the given wait value, and exits the function if the timeout is reached or if the passed channel's value is not equal to the wait value.

Syntax

SET <ReturnValue> msgwait(<message>, <channel>, <waitvalue>, <timeout>)

Parameters

message: message string to send to a mailslot, or a virtual string channel that contains the message to be sent to a mailslot

channel: VS or VO channel name in double quotes. This parameter must be a channel name. Customer and module names are invalid.

waitvalue: float or string value in double quotes. The msgwait function will wait while the channel value equals the waitvalue until the function times out.

timeout: time out time in seconds.

Return values

 0: The function exited normally, wait channel no longer equal to wait value.
-1: Time out waiting for wait channel to be set to something other than wait value.

Example

+0.1 SET xx msgwait("\\.\mailslot\Softpanel>>Hello from msgwait", "WaitChan", "3.4",5);

 

setbits¶

Sets the range of bits from high to low equal to the bit value specified (either 0 or 1). The setting occurs to the <channel> argument of the SET command.

Syntax

SET <ReturnValue> setbits(<high bit>,<low bit>,<bit value>)

Example

SET numSysBitChannel setbits(5,1,0)

* Where numSysBitChannel previously had a bitset performed on bit 5 and on bit 1.
* In this example, bits 5 to 0 of numSysBitChannel would be set to 0, and the value of numSysBitChannel would be changed from 17 to 0.
