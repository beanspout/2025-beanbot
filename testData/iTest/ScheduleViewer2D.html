ScheduleViewer2D¶

Runtime Operation¶

The following right-click options are disabled while iTest Console or AutomationPanel are running:
  • Add Column
  • Insert Column
  • Edit Column
  • Delete Column
  • Move Column Right
  • Move Column Left

Command Line Arguments¶

The ScheduleViewer2D command line only supports arguments with components in the following order:

  1. ScheduleName: Name of the schedule to load.
  2. Width: -1 to ignore.
  3. Height: -1 to ignore.
  4. Left: -1 to ignore.
  5. Top: -1 to ignore.
  6. Topmost: 1 - topmost/0 - non-topmost.
  7. Lock: 1-Lock/0 - unlock.
  8. StepChan: Name of virtual output channel for the current step number.
  9. ViewType: 1 - battery test/2 - battery.
  10. CircuitID: Circuit ID.
  11. MailslotNameChan: Virtual string channel name that ScheduleViewer2D sets the Mailslot name to be.

The argument components must be inputted in the order listed above and separated by a space to be interpreted correctly. Multiple instances of ScheduleViewer2D can run simultaneously. Each has a unique mailslot name (e.g., ScheduleViewer2D1 - ScheduleViewer2D99). If MailslotNameChan is given to ScheduleViewer2D, it sets the mailslot name to this channel. The value includes "\\.\mailslot\".

Example 1:¶

Launch a schedule as locked and non-topmost.

ScheduleViewer2D MyScheduleName -1 -1 -1 -1 0 1

Example 2:¶

Launch a test for circuit Cct03.

ScheduleViewer2D MyBattTest -1 -1 -1 -1 0 1 voStepChan 1 Cct03

Example 3:¶

Launch a test for circuit Cct03 with a mailslot name channel.

ScheduleViewer2D MyBattTest -1 -1 -1 -1 0 1 voStepChan 1 Cct03 vsMailslotName
