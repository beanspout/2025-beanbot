Overview¶

This guide provides support and details restrictions for using Python packages in the iTest development suite.

Packages¶

Python packages (*.whl) are files that can be used to add new APIs and extend the functionality of Python. By default, iTest includes several of these packages in the $SYSTEMDIR\PythonPackages folder; however, the user can add more if they need additional functionality in Python. When PythonEngine is run with the /installpackages command line option, it will attempt to install any Python packages found in the $SYSTEMDIR\PythonPackages folder, and then exit. The PythonPackages.log file will be updated with the status of each Python package installation.

When launching iTest, iTest will check if there are any packages in the $SUPPORTDIR\PythonPackages folder that do not currently exist in the $SYSTEMDIR\PythonPackages folder. If iTest finds any of these "missing" packages, then it will automatically run PythonEngine with the /installpackages command line option to install these packages before continuing to launch. In addition, any "missing" packages will be copied to the $SYSTEMDIR\PythonPackages folder, and the PythonPackages.log file will be updated.

Restrictions¶

This feature only applies when using the default Python installed as part of iTest. If the user sets the PythonHome setting in powertek.ini to a Python installation that is not the default, then PythonEngine does not attempt to install any packages there. The user must maintain that Python installation themselves.
