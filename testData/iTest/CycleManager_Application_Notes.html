Overview¶

This document contains Cycle Manager application notes for the Universal Transient Test. Regression formulas are obtained from the Electronic Code of Federal Regulations. In January of 2015, the Cycle Manager code was updated to reflect the new Maximum Test Speed calculations under CFR 1065.610 - 2014.

Executable Command Line Arguments¶

Passing “DEBUG” on the command line enables the output of the arrays used during calculations.

/autosc2¶

Run CycleManager in background to generate .sc2 output file.

Powermap File Requirements¶

To denormalize a normalized curve, a TCG power map file must be created. The following sections detail the optional and required items used to map this file:

Header Items¶

These items are located in the header and use the following format:

Syntax:

ParameterName=ParameterValue

Example:

IdleSpeedSpec=600

Required¶

  • Engine’s Specified Idle Speed
  • Engine’s Specified Rated Speed
  • File Rate (Hz)

Optional¶

  • The curb idle torque for the engine.
  • The time the TCG Map was created.
  • The date the TCG Map was created.
  • The test cell name where the TCG Map was created.
  • A data set identifier and can also be used as a description.

Logged Items¶

Required:¶

  • Speed Feedback
  • Torque Feedback

Optional:¶

  • Ambient Air Intake Temperature
  • Barometer
  • Dewpoint

Load Power Map button¶

This button loads the selected powermap file.

Learn Settings¶

Learn Options and Description

Option Description
Learn The amount of learning is selected by choosing ‘Apply % Learn’ and entering a percentage value. If a previous learn was used and you want to keep it the same, select Hold. If you want to reset the learn to zero, select Reset.
Reset Resets the Learn arrays to zero; no Learn is applied when arrays are generated. This is the default setting when Cycle Manager is loaded. Update the screen if Learn was previously applied.
Hold Maintains the previous percentage of Learn, regardless of the entries in the % entry fields.
Apply & Learn Causes the value in the % boxes to be applied to the error between reference and feedback. The resulting values are saved to the Learn arrays and added to the setpoint when those arrays are generated.

 

Note¶

  • The Hold option maintains the previous percentage of Learn regardless of the entries in the % entry fields.
  • Learn is an additive process. For example, assume arrays were generated using 50% Learn. If the UTT cycle is run but still does not pass statistical analysis, running Cycle Manager again and applying a 25% learn will result in a final setpoint value of the previous setpoint (which included 50% Learn) plus 25% of the currently observed error.
  • Learn arrays are re-created from logged data.
  • In order for Learn to work correctly, shift must remain constant.
  • The learning amounts are added to the denormalized speed and torque before the speed and torque setpoints are written
  • Learn values are NOT applied to idle points.
WARNING: In order for the learning function to work correctly, the newly created arrays must have the same shift as the previous arrays. This ensures the learning is applied to the correct values.

Analyze Previous Cycle Options¶

 
Previous Cycle Analysis

 

Options¶

  • Previous cycle filename: Data file from DAC system.
  • Cycle Analysis Options:
    • All Points: All points evaluated (default).
    • No Idle Points: No idle points evaluated.
    • Standard Point Deletion: Eliminates points if the torque setpoint is less than actual and the throttle is at 0% or the torque setpoint is greater than actual and the throttle is at 100%.

Report¶

Report components:

  • R2: Regression Line Analysis R2 value
  • SLOPE: Regression Line Analysis slope value.
  • INTERCEPT: Regression Line Analysis intercept value.
  • ERROR: Regression Line Analysis error of estimation value.

Speed Regression Criteria¶

ABSOLUTE (Intercept) > 50 = FAIL
Slope < 0.97 –or- Slope > 1.03 = FAIL
R2 > 0.97 = FAIL
Error > 100 = FAIL

Torque Regression Criteria¶

ABSOLUTE (Intercept) > 20.33 = FAIL
Error > Previous Cycle Max Torque * 0.13 = FAIL 
COLD:
Slope < 0.77 –or- Slope > 1.03 = FAIL
R2 > 0.85 = FAIL
HOT:
Slope < 0.83 –or- Slope > 1.03 = FAIL
R2 > 0.88 = FAIL

Power Regression Criteria¶

ABSOLUTE (Intercept) > 3.728 = FAIL
Error > Previous Cycle Max Power * 0.08 = FAIL 
COLD:
Slope < 0.87 –or- Slope > 1.03 = FAIL
R2 > 0.91 = FAIL
HOT:
Slope < 0.89 –or- Slope > 1.03 = FAIL
R2 > 0.91 = FAIL

Denormalization of Engine Speed¶

The denormalization of engine speed is calculated through the following formula:

Denormalization of Engine Speed Formula

 

Variable Descriptions

Variable Description
nnorm The normalized speed divided by 100.
nlo The lowest speed where the power is 55% of the maximum power.
npref The engine speed where the integral of the maximum mapped torque is 51% of the whole integral between nidle and n95h.
nhi The highest speed where the power is 70% of the maximum power.
n95idle The idle speed.
n95h The highest speed where the power is 95% of the maximum power.
nPmax The max power speed. Refer to the Replacement Process section for more information.

 

Replacement Process¶

In the events where the nhi and n95h values are not available, the following provisions apply:

  • nhi in the above formula is replaced with nPmax x 1.02.
  • n95h is replaced with nPmax x 1.02.
