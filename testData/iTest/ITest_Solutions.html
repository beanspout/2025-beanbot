Overview¶

Solutions are sets of configuration files that define the framework for a given test automation application. You can use SolutionBuilder and Test Manager to manage various aspects of your solution; refer to the Developer Tools documentation for more information.

By default, iTest includes Solution.Demo during installation. This solution demonstrates the use of major or new application features. For more information, refer to the Solution.Demo Quick Start Guide.

Solution Folder Structure and Requirements¶

When first launching iTest, the Select Support dialog displays. This dialog contains a list of available solutions and modules found in the $SYSTEMDIR. In order for a solution to appear in the Select Support dialog, it must adhere to the requirements as detailed in the Solution Folder Requirements documentation. New solutions can also be created from the Select Support dialog.

The solution folder is commonly known as the $SUPPORTDIR, and it is stored under the $SYSTEMDIR. For information about its structure, refer to the iTest Solution Folder Structure documentation.
