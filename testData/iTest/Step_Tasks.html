Overview¶

This document describes the Step Task columns in iTest schedules and master schedules. iTest schedules contain three step task columns while master schedules contain two step task columns. There are two types of step tasks: Conditional and Unconditional. Unconditional step tasks are only supported in master schedules; these step tasks are performed immediately once the step is entered.

Conditional step tasks are continuously evaluated in order from left to right when a new step is entered. The first time a step task's condition evaluates as true, Schedule Runner executes the action defined in the step task. At this point, the evaluation of the step task stops, and the schedule remains on that step unless an End Step or Go To Step action is triggered.

NOTE: Throughout iTest, step conditions may also be referred to as end of state (EOS) conditions.

Parts of a Step Task¶

Each step task consists of four elements:

  1. Description: This is a label that represents the step task. It is shown in the schedule spreadsheet to indicate what task is selected for a particular step task column. Note that this field is not available for master schedules.
  2. Condition: Conditions are referred to as step conditions. A step condition is a sequence of one or more expressions joined by a logical operator, which can be either AND or OR. Each expression consists of a parameter, followed by an operator, followed by an operand. Parameters and operands support simple expressions with only one operator; you can perform addition, subtraction, multiplication, and division. In addition, parameters and operands support the use of channels of all types except for virtual strings and virtual string arrays. For more information, refer to the Step Conditions documentation.
  3. Action: There are a number of pre-defined actions built into iTest. The selected action for a step task occurs when the condition entered above evaluates to true for the first time after a new step is executed. Actions are described in the Step Task Actions table below.
  4. Arguments: Actions have arguments that provide additional information about what the action is supposed to do. Actions and their arguments are described in the Step Task Actions table below.

For schedules, step tasks are created in SolutionBuilder's Step Task Check Setup dialog. To access the step task dialog, navigate to Tests > Schedules from the SolutionBuilder main menu. In a schedule, click a cell in a Step Task column to open the Step Task Check Setup dialog.
 
Step Task Check Setup Dialog

 
For master schedules, step tasks are created in SolutionBuilder's Edit Step Task dialog. To access this dialog, navigate to Tests > Master Schedules from the SolutionBuilder main menu. In a master schedule, click a cell in a Step Task column to open the Edit Step Task dialog.

Step Task Actions¶

The available step task actions and their arguments are described in the table below:

Step Task Actions - Descriptions and Arguments

Option Description Argument
Change Log Time Supports high-speed logging during the transient part of a step; it then changes to snapshot logs for the remainder of the step. New Log Time (in seconds)
End Step Stops the step immediately and proceeds to the next scheduled step. None
Go To Step Ends the current step and goes to the specified step number indicated in the dialog. Valid Step Number or Label
Increment Value Adds a constant value to the current value of a channel; subtracts if a negative value is used. Syntax similar to "set value": channel +1, where 1 can be any numeric value. To decrement: channel +-1 Channel + Value
Jump To Step Jumps ahead to specified step indicated in the dialog. Valid Step Number or Label
Pause Holds the test until a ResumeSchedule command is received. None
Pause GoToStep Resumes from Pause and the test goes to the user-entered step number rather than the next step. Valid Step Number
Save Value Writes the channel name and value to the circuit's INI file so the value can be restored when iTest is restarted. Alias Name
Set Value Sets a channel or parameter to a value with the syntax channel=value. To set multiple values simultaneously, separate the channel=value inputs by semicolons. value can also be a simple expression with operators +, -, * or /. For example,
Cntr1=Cntr2+Cntr3;Cntr4=Cntr1;Counter4=Counter2*6.
 
Note: String channels are not supported.		 Channel = Value
Start New File Supports breaking a potentially extremely large data file into smaller sections. It appends three digits to the current data file name starting at "001". Each time this action is executed the extender is incremented by 1. If this action is executed after the extender reaches "999", it results in an error message, and the extender is not incremented. None
Start Procedure Launches the procedure asynchronously (spawn). The procedure name can be entered directly or selected from a picklist. The Start Procedure picklist is defined in the picklist file set by the EOSActionPickList= powertek.ini setting (e.g., EOSActionPickList=SysActions).
 
Any necessary procedure arguments should be included in parentheses; for BTS, a special circuit index argument is typically required (i.e., <procedure name>($index$)).		 Procedure Name
Stop Test Stops the Schedule Runner from running the schedule. None
Start Log Starts the specified data log name of data log or name of Virtual String containing the name of the data log
Stop Log Stops the specified data log name of data log or name of Virtual String containing the name of the data log
Module Action* Sends a module action to the selected module(s). For more information, refer to the ModuleAction documentation. Module(s) and Action. Both are selected from predefined fixed lists.
System Action* Sends a system action to the selected module(s). For more information, refer to the SystemAction documentation. Module(s) and Action. Both are selected from predefined fixed lists.
Stop Master Schedule** Prematurely stops the master schedule at its current step. None
