Overview¶

The PythonEngine mailslot messages call scripts that are stored in the PythonScripts folder. The folder is supported in two locations: the support directory ($SUPPORTDIR) and the iTest installation folder ($SYSTEMDIR). When a mailslot message is sent, the support directory location is scanned first while looking for a script to execute. If the indicated script was not found, then the folder in the iTest installation folder is checked. PythonEngine currently supports the use of the mailslot messages detailed in the section below.

To add PythonEngine to your system, perform the following steps:

  1. Navigate to your $SUPPORTDIR.
  2. Open the powertek.ini file in a text editor.
  3. Add the following syntax to the Preload .ini setting:
     
    \\PythonEngine.exe>>
  4. Save and exit the file.

Python Module Specification¶

When calling a Python script, you must specify the Python module that contains the function to be executed. If your script file is Script1.py, then the module name for this is Script1; the file extension cannot be included in the module name.

Script files can be located in sub-directories within the PythonScripts folder. When specifying a module that is contained in a sub-directory, the sub-directories must be separated using the period (.) character, not a slash as is typical in Windows. For example, if your PythonScripts folder contains a sub-directory called Dir1, which contains a script file called Script1.py, then the module specification for this file would be Dir1.Script1.

Mailslot Messages¶

NOTE: You can use the tilde ( ) to escape characters in mailslot messages.

 

Mailslot Messages

Mailslot Message Description Syntax
DEBUG The DEBUG mailslot message is used to remotely debug a script running in PythonEngine. The Eclipse/PyDev debug server must also be running. This message executes a Python function in a module, where the message supports zero to ten arguments (either string or numbers). Any error messages that occur when loading the module or executing the function are written to Softpanel. 
DEBUG pythonModule pythonFunction(par1, par2, ... ,par10)
DIE Closes PythonEngine. 
DIE
SPAWN The SPAWN mailslot message executes a Python function in a module. Zero to ten arguments are supported; arguments can be strings or numbers. Any error messages that occur when loading the module or executing the function are written to Softpanel. 
SPAWN pythonModule pythonFunction(par1, par2, ... , par10)
