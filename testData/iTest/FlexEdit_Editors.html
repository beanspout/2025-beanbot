Overview¶

Editor displays are used to complete and perform tasks, they enable you to modify channel values to perform various calibrations or setting configurations.

AIM/AOM Editors¶

The FlexEdit AIM and AOM editors are used to modify and calibrate analog input and output channels within a solution. These editors enable you to modify the values of the channels for customized calibrations. When the modified calibration configuration is saved, the calibration equation updates the previously defined channel values to the channel values defined in the editor. The changes are immediately reflected in iTest whether the system is running or not.

Existing analog channels defined in your solution will display in their respective editor when you launch FlexEdit. The listed analog channels will only include uncalibrated channels, channels that have been calibrated using first order polynomial equations, or have their c2 - c6 coefficient values set to zero. Additional filtering is also available.

AIM - Analog Input Management¶

The Analog Input Management editor is used to calibrate analog input channels using an order 1 equation. You can modify the channel values for low voltage, high voltage, low EU, and high EU. Additionally, you can modify the customer name for each channel, its alias, as well as reported units and the decimal precision.

AOM - Analog Output Management¶

The Analog Output Management Editor is used to calibrate analog output channels using an order 1 equation. You can modify the channel values for low voltage, high voltage, low EU, and high EU to perform customized calibrations. Additionally, you can modify the customer name for each channel, its alias, as well as reported units and the decimal precision. The AOM editor contains all the columns described in the AIM editor, however, with the addition of the Source column which is used to link an analog output channel to another channel in the solution, modifying the originating signal value of the channel.

Customizing the AIM/AOM Editors¶

Limit Shown Channels¶

The AIM/AOM editors can be configured to limit which analog input and analog output channels are shown through use of the <checkuserdefined1> tag. When the tag value is TRUE, only analog input and analog output channels with a non-zero value in their User Defined 1 RDB field will be displayed; this RDB field can be edited in the Channel Definitions editor in Test Manager or SolutionBuilder.

Example:

<editor>
    <type>AIM</type>
    <checkuserdefined1>TRUE</checkuserdefined1>
</editor>

SolutionBuilder Channel Definition Editor: User Defined 1 Column

 
Configured AIM Editor Example

 

Modify Voltage Range¶

The AIM/AOM editors default to a min/max voltage range of -1200V - 1200V. The <minvoltage> and <maxvoltage> tags can be used to modify this range.

Example:

<editor>
    <type>AIM</type>
    <maxvoltage>10</maxvoltage>
    <minvoltage>0</minvoltage>
</editor>

 

AKA Editor¶

The AKA editor, also referred to as the Customer Name editor, is used to assign a channel to a customer name. Changes to the alias and display units for the channel can also be made. This editor supports the use of wildcard characters (* or ?) within the <file> tag of the FlexEdit.xml file. For more information, refer to the Using Wildcards section below.

Customer names are used as placeholders for channel names to provide greater readability and to eliminate the need to reconfigure portions of the iTest solution as hardware is migrated in and out. You can assign the channel to different customer names regardless if iTest is running; however, if iTest is running, the customer name cannot be referenced in any of the VCL script. If any customer names are referenced when saved, FlexEdit will not allow any of the new customer channel names to be applied.

You can right-click within the editor to insert, add, edit, or delete channels. Once you have added a channel, you can change its customer name, alias, or display units using the editor columns.
 
Customer Name Editor

 

Using Wildcards¶

Wildcard characters (* or ?) can be added to the <file> tag of the FlexEdit.xml file. This is used to automatically create a new AKA editor workflow task for each matching file found within the system. This eliminates the need to define separate tasks for each customer name file in the XML. The following restrictions apply to the use of wildcard characters:

  1. Files must be associated with the AKA editor.
  2. The wildcard character cannot be in the folder path to the file in the <file> tag; it can only be included in the filename. For example, C:\ADT\*\CustomerName.aka is not supported, but C:\ADT\iTest4.3\Cust*.aka is.
  3. The filename specified in the <file> tag cannot start with a wildcard character.
  4. The <duplicate> tag is not supported. This tag is used by the Duplicate and Delete right-click options. These options are disabled for tasks generated from a wildcard character even if the <duplicate> tag is specified for the task in the XML file.

When using a wildcard character to generate a task, you can use the $FILENAME$ macro in the following tags: <name>, <tooltip>, <help>, and <commandargs>. The $FILENAME$ macro is replaced with the filename of the file for the task.

The example below demonstrates how to use a wildcard character and the $FILENAME$ macro within the FlexEdit.xml file:

Example:

<task>
    <name>Wildcard $FILENAME$</name>
    <file>$SUPPORTDIR\DB\Customer\CustMax*.aka</file>
    <editor>
        <type>AKA</type>
    </editor>
</task>

Channel Attribute Editor¶

The Channel Attribute (CAV) editor is used to mark channels as being available for use in set value tables, step tasks, and circuits. These channels could be used, for example, as a step parameter, a step operand, or both in a Step Condition (EOS). In order for channels to appear in the CAV editor, the channel's User Defined1 RDB field must be greater than 1; for more information about this field, refer to its entry in the RDB Field Definitions documentation. The CAV editor can also be used to influence the listed channels in FlexEdit's AIM/AOM editors when they are using the <checkuserdefined1> tag; refer to the Customizing the AIM/AOM Editors section above for more information.

In the CAV editor, you can list additional channels for use, remove channels, update the channel's alias and description, and specify the channel's unit type and decimal precision. In addition, you can mark the channel as a step parameter and/or step operand as well as list it in a log header form.

To list additional channels, right-click on a row within the CAV editor and select the Add... option. The Channel Selection dialog displays, allowing you to select a channel. To remove a channel from the list, right-click on a row within the editor and select the Delete option.

NOTE: To open the right-click menu, the CAV editor must already list at least one channel.

Creating the Log Header Form¶

In the CAV editor, you can add/remove channel entries to a log header form via the Log Header column. The added entry is a read-only entry formatted as a string field that will appear in the left column of the form. It is labelled using the channel's alias.

Example Log Header Form Entry

The log header form written to is specified by the UserVariableListPath and LastUserVariableList powertek.ini settings (i.e., UserVariableListPath + \\ + LastUserVariableList). If the log header form does not already exist, then a new one is created.

For more information about log header forms, refer to the Data Logging documentation.

DLL Mapping Editor¶

The mapping editor contains two separate lists. On the left are the remote channel names from the import file (e.g., A2L file). This listing can be trimmed down by using a wildcard search in the Name Filter field similar to the filtering available in the Channel Selection dialog display found in many other iTest editors. On the right is a list of iTest channel names and the current mapping for each channel.

To map a device name to an iTest channel name, select one or more device channel names either by using normal list selection or by checking the checkbox. Next, select an iTest channel name. The first selected device name will be mapped to the first iTest channel name. If more than one device name is selected, then subsequent device names will be mapped to the next iTest channel.

Pre-Built DLL Plugin Specific Options¶

FlexEdit Asap¶

  • The FlexEditAsap editor contains a second Browse button that allows for the selection of the ACF file to be used. This may modify what is displayed in the left listing, as the A2L file name information is updated based on the selected ACF file.
  • The FlexEditAsap editor allows you to type in the fully-qualified path and filename for a Hex file that is to be used in conjunction with the selected A2L file. This field is updated whenever an ACF file is selected.
  • When an iTest channel is mapped to an ASAP channel, this editor reads the unit label from the A2L file and replaces the iTest channel's unit with the new unit. Unmapping a channel will cause the channel's unit to be cleared.
  • You can filter the input and output channels via the <inputchannelfilter> and <outputchannelfilter> tags. These tags allow you to auto-filter the list of iTest channels available for mapping based on the selected interface. For more information, refer to the <inputchannelfilter> and <outputchannelfilter> documentation. These filter results can be further pared down using the Filter field.

FlexEdit CAN¶

This editor allows you to update the scaling and other details of the CAN bus signals. It does not import the messages themselves.

  • When an iTest channel is mapped to a CAN signal, this editor reads the unit label from the DBC file and replaces the iTest channel's unit with the new unit. Unmapping a channel will cause the channel's unit to be cleared.
  • You can filter the input and output channels via the <inputchannelfilter> and <outputchannelfilter> tags. These tags allow you to auto-filter the list of iTest channels available for mapping based on the selected bus interface. For more information, refer to the <inputchannelfilter> and <outputchannelfilter> documentation. These filter results can be further pared down using the Filter field.

Left List

The import file contains the CAN signals available for use by the CanNeo bus interface. Separate .dbc files can be used for the input versus the output signals. The paths for each import file is stored in the FlexEdit .ini file for that device and bus (e.g., FlexEditCanNeo0_HS1.ini).

Example:

[Driver]
customerNameFile=CanNeo0_HS1.AKA
ChannelFile=$SUPPORTDIR\DB\Bcc1.FIL
lolFile=MAIN.LOL
lastInterface=2

[Interface1]
ImportFileName1=$SUPPORTDIR\CanNeoTestCases.dbc
ImportFileName2=$SUPPORTDIR\CanNeoTestCases2.dbc

lastInterface=1 selects the first bus interface (e.g., CanNeo0_HS1 - Input) on open.
lastInterface=2 selects the second bus interface (e.g., CanNeo0_HS1 - Output) on open.

Right List when SystemType=BTS

FlexEdit Indiset¶

This plugin uses two files: the import file and the mapping file. The import file lists the remote parameter names that can be accessed within Indiset. Two import file formats are supported: CSV and remote output definition. The mapping file links an iTest channel to a remote parameter name (i.e., a mapping).

  • When an iTest channel is mapped to an Indiset parameter, this editor reads the unit label from the import file and replaces the iTest channel's unit with the new unit. Unmapping a channel will cause the channel's unit to be cleared.

CAUTION: CSV files do not contain units. Using this style of import file will clear units from the iTest channel as part of both the map and unmap events.

The path to the import file is stored in FlexEditIndiset.ini.

[Driver]
customerNameFile=Indiset.AKA
lolFile=Indiset.LOL
lastInterface=1

[Interface1]
ImportFileName1=$SUPPORTDIR\ANDTest.txt
ImportFileName2=$SUPPORTDIR\ANDTest.txt

The path to the MAP file is stored in FlexEdit.xml using the <mapfile> tag. Use of a fully qualified file path is not permitted in the <mapfile> tag as FlexEdit automatically adds $SupportDir\\RemoteMaps\\ to the file name.

<task>
    <name>Indiset Mapping</name>
    <file>$SUPPORTDIR\config.drv</file>
    <editor>
        <type>Indiset</type>
        <dll>FlexEditIndiset.dll</dll>
        <mapfile>asap.map</mapfile>
    </editor>
</task>

Dynamic Calculation Editor¶

The Dynamic Calculation Editor enables you to edit the active running dynamic calculations file. When changes to this editor are made and saved, the CalcEngine process stops and restarts using the new changes made to the file. The calculations run independently and asynchronously to the real-time system because they are processed by separate executables. This allows for modifications while the file is actively running, without affecting the system or causing a full rebuild. Supported channel types include: virtual output, analog output, digital output, and PWM channels. An example of the dynamic calculation editor is displayed in the image below:
 
Dynamic Calculations Editor

 
The dynamic calculation editor Channel Name, Customer Name, Alias, and Units fields can all be edited as well as the actual formula running on the channel. For descriptions of the editor fields, refer to the table below. When you select a channel name, it is important to select a virtual output channel that has been pre-designated to be a container for the dynamic calculation values. This naming convention should come from the person that designed your solution.

When you save the dynamic calculations, the formulas and other fields are parsed for correctness prior to actually saving the file. Any errors cause a message to be generated in the error message tab of the message pane or in a separate dialog that describes any errors within the formulas. Any syntactical errors prohibit you from saving the dynamic calculations.

Additionally, you can quickly reorganize calculations in your editor by highlighting multiple rows and using the shortcut keys, CTRL+SHIFT+UP/DOWN, to move the rows in the proper direction.

Dynamic Calculation Formulas¶

Dynamic calculations support #if logic embedded into the formula string for an individual calculation. The syntax is as follows:

#if (condition) \ntrue_result\n#endif

or

#if (condition) \ntrue_result\n#else\nfalse_result\n#endif

Where: "\n" delimits the statements

CAUTION: #elseif statements and nested #if/#else statements are not supported.

INI Editor¶

INI editors are often utilized to prompt operators for configuration information to prepare for tests. The editor contains two columns, the primary column indicates the property (or channel name) and the second contains the value assigned to the channel. The values entered in this editor can be saved to an iTest .ini file as well as the RDB, dependent upon the FlexEdit design. INI editors are often divided into collapsible sections to better organize information.
 
INI Editor Example

 
The values in the Value column of the INI editor can be designed to accept different forms of acceptable input, such as:
  • Numeric: Entry field type that may have restrictions on length and precision.
  • Text: Entry field type that may have restrictions on length and character sets.
  • Picklist: A list of pre-defined options that can be selected.
  • Combobox: Field that enables you to select from a list of pre-defined options or enter a value.
  • Checkbox: Boolean function that writes a value of 0 or 1.
  • Channel: Displays the Channel Selection dialog.
  • File: Displays a Windows File Explorer dialog to select a file from a defined location.
  • Folder: Displays a Windows File Explorer dialog to select a folder from a defined location.

This must be a numeric value; the default value is 0.

Fixed Column¶

The Limits editor supports the ability to define a fixed column. When this feature is set, you can scroll through the editor horizontally, while maintaining the view of the fixed column. For more information, refer to the Global Editor Settings documentation.
 
Fixed Column

 

Log Order List Editor¶

The LOL editor enables you to select and add channels to a specific Log Order List file. You can use the editor to modify the name of the channel that is logged, activate channel statistics for average logs, or log data quality settings for channels with data quality enabled. The right-click menu options for managing channels are also available in this editor enabling you to add, delete, or re-arrange channels in the list.

NOTE: Changes to Blackbox.lol requires an iTest restart. FlexEdit will not update these during runtime.

 
LOL Editor

 

Additionally, in this editor, you can:

  • Click the column header label to sort/reverse sort the list alphabetically.
  • Double-click the channel name value to display the Channel Selection dialog.
  • Drag and drop channels from the Helper Bar into the LOL editor. The channel's Customer/Module Name and Alias fields are automatically populated if applicable.
  • Highlight multiple rows and use the shortcut keys, CTRL+SHIFT+UP/DOWN, to move the rows in the proper direction.
  • View available Customer/Module Names and Alias for channels. These fields are read-only.
  • Copy/Paste text using the shortcut keys CTRL + C and CTRL + V.

Schedule Editor¶

FlexEdit has the ability to display and edit test schedules using an SC2 schedule editor. You can add new rows or test steps to the schedules as well as update the values, however, you cannot modify the structure of the columns or change the descriptions for the columns, these are pre-configured by the solution designer.
 
Schedule Editor

 

Channel Property Change Dialog¶

The Channel Property Change dialog allows you to specify the appropriate units and the decimal point precision for a channel in the AIM, AOM, and Dynamic Calculations editors.

To launch the Channel Property Change dialog, double-click the Units column for a channel, or right-click a column and select Edit Property*. The Units Label and Decimal Points are the only editable fields in this dialog. You can type the appropriate units label in this field or select a unit type from the drop-down. Define the decimal point precision using the drop-down's available options.
 
Channel Property Change Dialog

 

* This option is only available in the Dynamic Calculations editor.
