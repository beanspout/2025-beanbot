Overview¶

This document describes the iTest installation folder and its sub-components. When you install iTest, it is suggested that the top folder be called ADT. The iTest documentation refers to the installation folder as C:\ADT\iTestX.X where X.X is the version number of the iTest application. The following sections focus on the primary folder structures $SYSTEMDIR and $SUPPORTDIR.

Terminology¶

iTest Macro Descriptions

Term Definition
$SYSTEMDIR This is the folder where iTest is installed on the PC.
 
Example: C:\ADT\iTest4.1\
$SUPPORTDIR This is the solution folder.
 
Example: C:\ADT\iTest4.1\Solution.Demo\
$DATADIR This is the folder that contains all logged data.
 
Example: C:\ADT\iTest4.1\Data\
$EXECUTEDIR This is the folder that contains iTest executable files and other binary files.
 
Example: C:\ADT\iTest4.1\Execute\
$PYTHONDIR This is the folder that contains Python support files including the supported libraries.
 
Example: C:\ADT\iTest4.1\Execute\Python37\

 

$SYSTEMDIR Structure¶

$SYSTEMDIR or the system folder is the location on the Windows PC where iTest was installed. Typically, this is a folder like C:\ADT\iTestX.X where X.X is the version of iTest. You can install multiple versions of iTest on the same machine, so it is not uncommon to see the C:\ADT directory have multiple sub-folders such as \iTest3.6, \iTest3.7, etc.

NOTE: The iTest $SYSTEMDIR and $SUPPORTDIR paths may not contain spaces.

The following folders are located under $SYSTEMDIR:

$SYSTEMDIR Folder Descriptions

Folder Name Description
Art This folder contains icons, bitmaps, and other image files that are used by some iTest programs and can be useful when designing applications.
Data This is the default location for project data files ($DATADIR). Alternatively, you can use the $SUPPORTDIR\Data folder.
DriverLibrary The DriverLibrary folder contains all of the standard drivers and driver support files that are available with iTest. By default, all drivers are automatically installed during iTest installation, but you must have the proper dongle to use a specific driver.
Execute This folder is also referred to as $EXECUTEDIR. It contains all the binary files required for iTest such as - .exe files, .ocx, .dll and more. The folder also contains the LabAgent web pages.
Help This folder contains the driver help files in PDF format.
Installation This folder contains support files and sub-folders used by all solutions in iTest.
Solutions This is the $SUPPORTDIR folder. For example, Solution.Demo, Solution.Battery, or any other name used to identify your solution. You can have multiple solutions in the $SYSTEMDIR.
Transaction The transaction files used for Part Tracker are stored locally here before they are uploaded to LabCentral.
TestSystemDiag The Test System Diagnostic files are stored locally here before they are uploaded to LabCentral.
Utilization The Utilization data files are stored locally here before they are uploaded to LabCentral.

 

$SUPPORTDIR Structure¶

$SUPPORTDIR or the solution folder is the location on the Windows PC where your specific application is contained. Typically, this is a folder like C:\ADT\iTestX.X\Solution.YYY where X.X is the version of iTest and YYY is the name of the solution you are developing. Solution names may also indicate the specimen type for which you are creating an application. Multiple solution folders are permitted in $SYSTEMDIR.

The following folders are located under the $SUPPORTDIR:

$SUPPORTDIR Folder Descriptions

Folder Name Description
Art Icons, bitmaps, and other image files that are used by this solution.
Calculations Contains sub-folders for each calculation rate (1Hz, 10Hz, 100Hz, SystemRate), which in turn contains the individual calculation (.dat) files.
DATA This folder contains logsetup.dat and the (.lol) files. If this is the location of $DATADIR, it will also contain the project subfolders for datalogs.
DataQuality Contains the files for all data quality groups (.dqg).
DB Contains the channel definition files (.fil)
DigitalOutConds Contains the files for Set Value Tables (.dco)
Documentation Any existing documentation for the solution.
eGateCfg Contains dynamically created temporary files for use when you have an egate or QPac controller.
EOSConditions Contains files for Step Conditions. (.eos)
EXECUTE Contains the sub-folders for the 3 display types available in iTestConsole - Display=Main, Custom=Dialog, Right=Auxiliary Monitor
FlexEdit Contains the solution's FlexEdit.xml files.
Layouts Contains the solution's default and saved layouts for AutomationPanel.
Limits Contains files for Limit Groups (.lim)
Logs Contains the program log files for the various iTest executables. These files are mostly used for forensics when a problem occurs.
Maps Contains the map's (2D Table) files (.2dt)
MasterSchedules Contains the solution's master schedule (.msf) files.
Modules Contains some housekeeping files for each module that has been added to the solution.
Picklist Contains picklist files (.pkl)
pid Contains folders for each individual PID.
PlotTemplates Contains saved plot layouts created from DataMiner. To access DataMiner, refer to the customer portal at support.aanddtech.com then add the DataMiner file to the $EXECUTEDIR folder.
PythonScripts Contains the solution's python script (.py) files.
RCS Contains previous versions of all saved files from Test Manager if file RCS is enabled.
Reports Contains .rpp and .rpt files for iTest Reports.
Schedules Contains all schedule files (.sc2)
Sequences Contains sequence (.seq) files.
SetValues INI files for system values that can be set. E.g. limit values.
Solution Contains some temporary files used for editing the logsetup.dat file and the internal variables file when you have modules in the system.
Specimen Contains the INI files for the units under test.
StringTables Contains translation files for your application.
Test Contains all if the procedure (.tsc) files in various sub-folders.
TestFormula Contains the formula (.fml) files for the solution.
Toolbar Contains test control toolbar files (.tctb)
Varlists Contains the User Variable list files (.uvl)
VXI This is a legacy folder that is used for many of the dynamically created files when iTest starts up. These are typically transferred down to the RTOS. In addition, it contains files for the file based arrays.
