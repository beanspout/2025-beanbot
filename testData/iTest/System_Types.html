System Types¶

The following is a list of supported powertek.ini system types and their use.

Option Use
IBS The default system type for use with most solutions intended to run on Windows only such as Simulation, iTest.VSA, and others.
RTX_DIST For use when the GUI is running on Windows and a separate system for doing real-time control; this is typically an Advantech PC from A&D running INtime. RTX_DIST always assumes INtime is the remote real-time OS.
 
This system type can also be identified as a battery system if BatteryAppUsesArrays=TRUE in the powertek.ini file.
BATTERY System type for battery test systems using iTest.BTS, must also have BatteryAppUsesArrays=TRUE.
ADXPRTS_DIST For use when the GUI is running on Windows and a separate system for doing real-time control; this is typically a Dell embedded PC 5000 or an A&D Procyon running Xenomai/Linux. ADXPRTS_DIST always assumes Xenomai/Linux is the remote real-time OS.
Phoenix For use when iTest is being used as the GUI for a Phoenix/C3 data acquisition box. Most commonly utilized for combustion analysis.
WIN_DIST For use when the GUI is running on Windows and real-time control is done on Windows. WIN_DIST always assumes INtime is the remote real-time OS.
 
This system type can also be identified as a battery system if BatteryAppUsesArrays=TRUE in the powertek.ini file.
