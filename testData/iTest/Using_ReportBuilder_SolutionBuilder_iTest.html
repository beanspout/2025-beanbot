Overview¶

SolutionBuilder allows for the creation of web report templates which when coupled with LabAgent provides a tool for visualizing information in data files. Report objects can be configured to display the contents of data files. When data files are loaded, their contents populate the report. This functionality enables you to reuse templates by simply changing the loaded data files. Only .dat formatted data files are supported. In addition, SolutionBuilder uses the .wrpt file extension for web report templates and stores the files in the $SUPPORTDIR\WebReports folder.

NOTE: SolutionBuilder does not support web report templates provided from LabWorX with the .xml extension.

Report Objects¶

Report objects are the different shapes and tools you can add to the layout. There are different report object types such as A&D-specific objects, arrows, and clipart. There are also objects that allow you to create process models and flowcharts. The table below describes each of the report object types. To add them to the layout, select an object and drag and release it onto the layout.

Format Panel¶

You can use the format panel to modify page settings and object properties. When you select an object on the layout, three tabs display in the format panel: style, text, and arrange. Each tab contains various object properties which you can use to customize the objects in your report. The settings in the panel depend on the object that is selected.

When you use the format panel to modify an object's text properties, the changes apply to the entire object. However, you can limit text property changes to only highlighted text. Note that if the entire object's text is highlighted, then the font size property only applies to the first line of highlighted text.

Plot Object Use with Multiple Data Files¶

The use of a Plot object with multiple data files requires special consideration when using a channel for the X-axis. Unless using a defined range for the X-axis, the Y-axis channels will determine the data log files that the X-axis range is fitted from.

If the X-axis channel is in a source data file for a Y-axis channel, then the X-axis minimum and maximum range will be adjusted to fit with the corresponding Y-axis data points. If a Y-axis channel's source data file does not also include the X-axis channel, then the X-axis channel data points will not be used to determine the X-axis range for the Plot object. Instead, the epoch time column data points will be used.

Data Allows you to select a value from the data file loaded to ReportBuilder. You can select either the first or last value as well as a value from a specific row. The row is specified in the Row Index field.
Row Index The row in the data file to pull the value from. A positive number counts rows from the top of the data file, and a negative number counts rows from the bottom of the data file. For example, if the number 5 was entered, then the fifth row from the top of the data file will be used to pull the value. Value State value or string that will cause the set color and string to display (i.e., the state of the indicator). This field supports the use of channels and macros, allowing you to dynamically control the states based on the loaded data. If a channel is referenced, then the specified channel will be read from the same row that the indicator's source is reading from. If using a macro (e.g., %<voChannel01>%), then values can be read from different data rows, attributes, or header values. For more information, refer to the Using Macro Substitutions section.
 
For strings, a case-insensitive comparison is used (e.g., helloworld=HelloWorld). If a match is not found, then the indicator displays a gray background.
 
For numeric values, to determine the indicator's state, the channel's value is checked against each state value to find the closest state value that is also >= to the channel value. For example, if the indicator's properties are set where a state value of 9 indicates the color "red" and 10 indicates the color "blue," then a channel value of 9.9 would indicate the color "blue" since 10 >= 9.9 > 9.

Using Macro Substitutions¶

ReportBuilder provides support for macro substitutions. These substitutions can be used in text, title, indicator, or shape objects to display the values of channels, header values, or a timestamp.

The syntax for a macro substitution is shown below:

%<ChannelName>%

Indices¶

You can include index values in the macro substitution to display a specific data value from the data file. Positive index values read rows from the top of the file. Negative index values read rows from the bottom of the file. For example, if you use the number 6, the data point in the sixth row from the top of the file is displayed.

If the index value references a non-existent row in the file, then the macro will resolve to an empty string. For example, if a data file has 5 rows, and your macro is @%<ChannelName:6>%, then the macro will resolve to an empty string. If the channel does not contain any data rows (e.g., invalid channel name or truncated data file), then the macro will resolve to an empty string.

Syntax¶

%<ChannelName:number>%

Example¶

In this example, the channel value in the eighth row from the bottom of the file is displayed.

%<voDmo1Feedback:-8>%

Timestamps¶

The %<CURRENTDATETIME>% macro can be used to show a timestamp on a generated report. This timestamp is based on the date/time on the machine used to generate the report. By default, the %<CURRENTDATETIME>% uses the following date format: MM/dd/yyyy HH:mm:ss. You can specify a date format using the following values; these are case-sensitive:

NOTE: You must select a data file in order to use this macro. The dash ( - ), slash ( / ), colon ( : ), and space characters are supported.
  • yyyy - 4 digit year
  • yy - 2 digit year
  • MM - 2 digit month (01 - 12)
  • dd - 2 digit day of the month
  • hh - 2 digit hour (00 - 11)
  • HH - 2 digit hour (00 - 23)
  • mm - 2 digit minute (00 - 59)
  • ss - 2 digit seconds (00 - 59)
  • aaa - AM or PM

Syntax¶

%<CURRENTDATETIME:DateFormat>%

Examples¶

%<CURRENTDATETIME>%
%<CURRENTDATETIME:MM/dd/yyyy HH:mm:ss aaa>%
%<CURRENTDATETIME:MM-dd-yyyy HH:mm:ss aaa>%

Generating PDF Reports¶

LabAgent can generate PDF reports using a local data file via its WebReport feature. Web report templates are configured in ReportBuilder then run using a LabAgent mailslot message. Alternatively, web report templates can be configured in LabCentral and then the template file downloaded to the $SUPPORTDIR\WebReports folder from the WebReports folder in Manage Lab Files.

To generate the PDF report, send the $WEBREPORT mailslot message to LabAgent as specified below:

NOTE: A third-party tool must be used to view generated PDF files.

$WEBREPORT

Syntax:

 $WEBREPORT;<ReportName>;<DataFilePath>;<PDFFilePath>;<StatusChannel>

Arguments:

<ReportName>: Name of the report template file located in $SUPPORTDIR\WebReports with optional file extension. .XML and .WRPT files are supported. If no extension is provided, then .XML will be used. If no .XML file exists, then .WRPT will be used.

<DataFilePath>: The path to a data or image file to use in the report. Multiple data/image files can be used in the report by specifying a folder or a comma-delimited list; note that only one report will be generated. The file path can be a fully-qualified path or a relative path under the DRHoldForReviewDir powertek.ini setting. Note that previewing reports via SolutionBuilder's Preview option are restricted to data files under the $SUPPORTDIR regardless of the specified file path. Macro substitution is supported (e.g., $SYSTEMDIR or $SUPPORTDIR). If the specified file path is a folder, then the files are processed in alphabetical order. In the event of a channel name appearing in multiple files, data files are prioritized in the order they are listed; the data from the file with the highest priority is used in the report.

<PDFFilePath>: Full path to the destination PDF file.

<StatusChannel>: (Optional parameter) Name of channel to update with error/success code from report generation.

Example of Using a Single Data File

$WEBREPORT;MyReport;TestData.dat;C:\ADT\MyReport.pdf 
$WEBREPORT;MyReport;C:\ADT\iTest4.3\Data\PR-001\TestData.dat;C:\ADT\MyReport.pdf

Example of Using Multiple Data Files - Folder

$WEBREPORT;MyReport;C:\ADT\iTest4.3\Data\PR-001;C:\ADT\MyReport.pdf

Example of Using Multiple Data Files - List of Files

$WEBREPORT;MyReport;TestData1.dat, TestData2.dat, TestData3.dat;C:\ADT\MyReport.pdf;Counter1

In this example, TestData1.dat is given priority followed by TestData2.dat then TestData3.dat.

Error Handling

If an error occurs during the report generation, it will be reported to the LabAgent log file as well as to the Softpanel message queue. If a status channel is specified, it will be updated with one of the following values:

Status Code Descriptions

Status Code Description
0 Report successfully generated.
1 Invalid message format.
2 Invalid report name.
3 Invalid data file name.
4 Problem writing to the .pdf file.
