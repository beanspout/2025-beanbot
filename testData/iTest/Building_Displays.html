Overview¶

iTest allows users to create on-screen displays, in which display objects can be added to view and manipulate channel data. Users can create the following types of displays:

  • Standard Display
  • Dialog Display
  • Right Display

To access displays, expand the Presentation folder in the Test Manager tree, and then expand the Displays folder. The folders for each display time are visible, as well as folders for Templates and Themes.

Creating a new Display¶

To create a new display, complete the procedure below:

  1. With the Displays folder expanded, expand the folder name of a display to add (e.g., the Standard folder).
  2. Right-click the folder and select New <folder name> Display. A system-generated display name appears in the tree.
  3. Right-click the name and choose the Rename option.
     
    Rename New Display

     
  4. Right-click in the Test Manager Display Area and select Display Properties to open the Display Properties dialog box in which general display properties can be configured.
  5. The Size & Position tab displays the default display size. To change the display size, enter the new height and width values in the Size fields.
     
    Size and Position Tab

     
  6. Click the Color tab to configure the color of the display.
     
    Color Tab

     
  7. Click Select to open the Color dialog box in which a predefined background color can be selected or create a custom background color for the display. Click Browse to display the Open dialog box from which a bitmap can be selected to use for the display’s background. Bitmaps are imported at the size they were created and cannot be resized on the display.
  8. Click the Grid tab. The options on this tab allow you to show a grid of dots on the display to help you align controls. The grid does not display in the Console. Show Grid is selected by default. To change the size of the grid, enter new values (in pixels) in the Width and Height fields. Controls will Snap to Grid marks by default. If you don’t want controls to snap to the grid, deselect this option.
     
    Grid Tab

     
  9. Alternatively, you can send some messages when the display opens and/or closes. To do this, click the Mailslot Messages tab. This will bring up the following screen which can be used to specify a mailslot message to send when the disply opens and one to send when the display closes.
     
    Mailslot Messages Tab

     
  10. Click Apply to save changes without closing the Display Properties dialog box. Click OK to save changes and close the Display Properties dialog box.
  11. Next, select an object from the Object Bar drop-down list and click Insert, or click one of the display objects on the Display toolbar to insert an object of your choice onto the display.
     
    Select Object

     
  12. Right-click the object to open its properties dialog box, and specify the appropriate settings.

For complete details on each display type, including a general description, field names/definitions and field properties, see the "Types of Display Objects" section further down on this page.

Standard Display¶

Standard displays are visible in the display area of iTest Console. They are accessible from the Command Bar, from a mailslot message to softpanel, or by selecting Displays/Standard from the toolbar.

You can use the following VCL to display a Standard Display:

"\\.\mailslot\Softpanel>>$UPPERDISPLAY;MyDisplay.dsp"

To create a new Standard display, complete the steps in the "Displays Overview" section above.

Dialog Display¶

Dialog displays are custom-sized displays that are only accessible by using a mailslot message to the Softpanel. Dialog displays are floating windows with definite size and starting positions. Dialogs are typically used to notify an operator of some exceptional event, or to retrieve information from an operator. They are also used to provide task-specific controls.

NOTE: Size and position of Dialog Displays will be overridden at runtime if CustomDisplayShowLastPosition is enabled in the solution's powertek.ini.

For more information about the CustomDisplayShowLastPosition powertek.ini setting, see ITest INI Options

You can use the following VCL to display a dialog:

"\\.\mailslot\Softpanel>>$CUSTOMDISPLAY;MyDialog.dsp"

Where MyDialog.dsp is the name of the Dialog display.

In addition to initializing dialogs using VCL, action buttons can be created on the toolbar to initialize them.

To programmatically close a dialog (for example, from a button on the dialog itself), send the DIE message to the display.

"\\.\mailslot\MyDisplay>>DIE"

This technique also works to close standard and right displays.

To create a new Dialog display, complete the steps in the "Displays Overview" section above.

Right Display¶

The Right Display is typically a full size display that should fill the entire screen of an auxiliary monitor. In the past the auxiliary monitor was placed to the right of the operator console, hence the name Right Display. More than two monitors are supported, so Right Display does not necessarily have to be on the right. These displays appear on the Console if the Global Setting for number of monitors is greater than 1. Define the starting position and size using the display properties dialog. Starting position is relative to the 0,0 coordinate of the primary monitor defined in Windows.

To create a new Right display, refer to the Creating a Right Display document.

NOTE: Creating a Right display is only applicable when employing a multi-monitor configuration.

Themes and Templates¶

Themes

Creating a Display Theme¶

  1. Expand the Presentation folder from the Test Manager Tree, followed by the Displays folder.
  2. Expand the Themes folder.
  3. Right-click Themes, and then select New Themes Display. The Themes dialog box displays in the Test Manager Display Area and a new theme named Theme1 is added to the Test Manager tree.
     
    Themes Dialog Box

     
  4. Select a Background Color for the display from the list of colors. Select Custom Color from this list to create a custom color.
  5. Select a Background Color, Foreground Color, and Font settings for Objects - General. Select Custom Color from this list to create a custom color.
  6. Click Apply to All Objects to apply these background and foreground colors to all individual controls. The selected colors will display in the BackColor and ForeColor fields for the individual controls.

Applying a Display Theme¶

  1. Expand the Presentation folder from the Test Manager Tree, followed by the Displays folder.
  2. Double-click on the display to which you want to apply a theme.
  3. Expand the Themes folder in the Tree Bar.
  4. Drag the theme you wish to apply from the Tree Bar onto the Test Manager Display Area. The display as well as the controls on the display will change to the colors specified by the theme.
  5. Click File > Save to save the updated display.

Using Templates¶

Templates are preconfigured controls or groups of controls that can be used as starting points for new displays. Template displays are particularly useful when a user has a control that they wish to use over and over again. In this case that user would create a template and then apply it to a new display.

Templates

Creating a Display Template¶

  1. Expand the Presentation folder from the Test Manager Tree, followed by the Displays folder.
  2. Expand the Templates folder.
  3. Right-click Templates, and then select New Template Display. A blank display opens in the Test Manager Display Area and a new display named Template1 displays in the Tree Bar.
  4. Right-click the new display, select Rename, type a new name, and then press ENTER.
  5. Right-click in the Test Manager Display Area and select Display Properties to open the Display Properties dialog box in which you can configure general display properties.
     
    Size and Position Tab

     
  6. The Size & Position tab displays the default display size (1024x460). To change the display size, enter the new height and width values in the Size fields.
  7. Click the Color tab.
     
    Color Tab

     
  8. Click Select to open the Color dialog box in which you can select a predefined background color or create a custom background color for the display.
  9. Click Browse to display the Open dialog box in which you can select a bitmap to use for the display’s background. Bitmaps are imported at the size they were created and cannot be resized on the display.
  10. Click the Grid tab. The options on this tab allow you to create a grid of dots that displays on the display to help you align controls. The grid does not display on the Console.
     
    Grid Tab

     
  11. Show Grid is selected by default. To change the size of the grid, enter new values (in pixels) in the Width and Height fields.
  12. Controls will Snap To Grid marks by default. If you don’t want controls to snap to the grid, deselect this option.
  13. Click Apply to save changes without closing the Display Properties dialog box. Click OK to save changes and close the Display Properties dialog box.
  14. Select an object from the Object Bar list and click Insert or click on one of the controls on the Display toolbar to insert a control on the display.
  15. Right-click the control to open its properties dialog box, and specify the appropriate settings.
  16. Click File > Save to save the template.

Applying a Display Template¶

  1. Expand the Presentation folder from the Test Manager Tree, followed by the Displays folder.
  2. Expand either the Standard, Right, Softpanel or Dialog folder.
  3. Right-click Standard, Right, or Dialog and then select New Standard Display, New Right Display, or New Dialog Display. A new display name (Display1, Right1, or Custom1) appears in the Tree Bar.
  4. Double-click the new display in the Tree Bar to open it in the Test Manager Display Area.
  5. Expand the Templates folder in the Tree Bar.
  6. Click the template you want to apply to the open display, and drag it onto the Test Manager Display Area. The controls in the template are now applied to the display.
  7. Make changes to the display as necessary.
  8. Click File/Save to save the updated display.

Editing Display Objects¶

Once you have placed a display object on the Display Area, you can resize, move, or delete it.

Resizing Objects¶

After you place a display object on a screen, you can resize it using the sizing handles.

Sizing Handles

The middle handles on the top and bottom resize the height of the object. The middle handles on the left and right resize the width of the object. The handles in the four corners resize both the height and width of the object.

  1. Place your cursor over the sizing handle in the direction you want to resize the object. The cursor changes to an arrow that indicates the directions in which the handle can resize the object.
  2. Click and drag the sizing handle in the direction you want to resize the object. Release the mouse button when the object is the desired size.
  3. Repeat this process for each side of the object that you want to resize.

Moving Objects¶

There are several ways in which you can move objects once they have been placed on the Display Area:

  1. Click and drag an object, or a group of objects, to its new location.
  2. Select an object or a group of objects and press the up, down, right, or left arrow keys on the keyboard. Each time you press an arrow key, the object moves one grid space as defined in the Grid tab on the Display Properties dialog box.
  3. Select an object, right-click, and select Move to Front. This places the object in front of other objects that are in the same location. Selecting Move to Back places the object behind all other objects.

Moving/Editing Groups of Objects¶

You can move or delete objects individually or as part of a group. You can also draw a box around a group like you can in Windows.

You can also select multiple objects in the display editor and then set common properties for each. For example, you can set the colors or fonts for a group of gauges or bar objects at the same time. Select the group you want to edit, right-click to select Generic Properties, and the following dialog will appear:

Generic Properties Right-Click Option

 
Generic Properties Dialog

This will allow you to set a consistent size, basic color scheme, font and border type to the selected objects. The Width and Height setting must be explicitly selected to override all of the current object sizes by also checking the Apply buttons for each.

Deleting Objects¶

Right-click the object or group of objects and then select Delete.

Editing Object Properties¶

  1. Right-click an object and select Properties, which displays the properties dialog box associated with the object.
  2. Make changes to the object and then click OK.

Undo/Redo Capabilities¶

iTest allows you to undo and redo your actions in the Display Editor. Any change to a display can be undone, and the default depth for undo is 40 actions. The default depth can be changed by modifying the UndoDepth setting in the Powertek.ini file.

To undo or redo an action, simply right-click in the grid area of the Display Editor and choose the appropriate option. The Ctrl-Z shortcut key is supported for the undo option.

Array Channels in Displays¶

There are three different ways to use array channel names in the display objects of a display.

  1. Static Index. When you configure a display object to display an array channel, you can add a static index to the end of the channel name (e.g. arrayval[2], arrayval[3], etc). When you do this the display object will always display the value from that index of the array, regardless of any index that the display might have been launched with.
  2. No Index. When specifying an array channel in a display object, you can also choose to not specify an index. You just use the name of the array channel. This has special significance and should only be used when you intend to launch an Indexed Display screen (see next bullet).
  3. Indexed Display. When you launch a display using a softpanel mailslot message, you can also add an index to the name of the display to make it an Indexed Display. For example, if you had a display called MyDisplay you could launch it with an index using either of the following mailslot messages:
    $UPPERDISPLAY;MyDisplay[2]
$UPPERDISPLAY;MyDisplay[2].dsp

All mailslot messages for launching displays support this syntax. This sets an index within the display that can be used by any of the display objects. Only display objects which were configured to display an unindexed array channel will use this display index. Display objects which use non-array channels or array channels with a static index will ignore the display index. When a display is launched with an index, the display title will change to show the index. The mailslot name of the display will also contain the index.

For example, you could create a display called MyDisplay which contains the following:

  • A Gauge where the channel name is set to arrayval
  • An Indicator where the channel name is arrayval[1]
  • A Bar where the channel name is HS1_SETPOINT.

If you sent the message $UPPERDISPLAY;MyDisplay[2] to the softpanel then the display would be launched and the title would show MyDisplay[2].

NOTE: A defined Display Label under the Display Properties dialog will supersede this title change.

The gauge would then display the third value of arrayval because it doesn't have an index and would use the display index the display was launched with, which was 2 in this case. The indicator would display the value of the second value of the array channel because it contained a static index (the [1] part). The bar would just ignore the display index and show the normal value of HS1_SETPOINT because HS1_SETPOINT is not an array channel. To close the display, you would send a DIE message to the mailslot name \\.\\mailslot\MyDisplay[2].

Because the mailslot name contains the array index, you can launch multiple copies of the same display at the same time with different indexes.

If you don’t use an index when launching a display and the display contains objects configured with an unindexed array name, then that object will display the value of the first element of the array.

Each display also has a mailslot command that allows you to change the array indices being referenced in the display:

$SETARRAYINDEX <new index>

Example: \\.\\mailslot\MyDisplay[2]>>$SETARRAYINDEX 5

This message can be sent to displays to change their array index. Doing so will also change the mailslot name of the display. So if the display file is test.dsp, and it was launched as test[2].dsp, it would have a mailslot name of \\.\mailslot\test[2]. Sending it as $SETARRAYINDEX 3 would change its index to 3 and the mailslot name would become \\.\mailslot\test[3].

A macro has been added to both the ADTButton and ADTCombo display objects to help send this variable-based mailslot. When constructing a message with these objects, use the macro $DISPLAYMAILSLOT$ to represent the current mailslot name.

Specifying Startup Displays¶

You can select the displays that automatically open when the Console starts. You can designate one Standard, Right, and Dialog display as a startup display. You cannot designate multiple displays of the same type as startup displays. For example, you can specify both a Standard display and a Right display as startup displays, but you cannot choose two Standard displays as startup displays.

  1. Expand the Presentation folder in the Test Manager Tree, followed by the Displays folder.
  2. Expand either the Standard, Right, or Dialog folder, depending on the type of display that you want to specify as a startup display.
  3. Right-click a display file and then click Startup Display. The display’s file icon in the Tree Bar changes to light blue to indicate that it is now a startup file.
     
    Startup File

     
  4. To change the startup file for a specific display type, right-click another file and then select Startup Display. The new display becomes the startup display, and the file icon for the previous startup display reverts to gray to indicate that it is no longer the startup display.

Line Drawing on Displays¶

When you right click in the display editor now, there is a new option for Draw Line.

Draw Line Option

When you select this option and release, the hand icon will appear. Move the mouse to where you want to begin the line and then hold the left mouse button to draw the line.

Draw Line

To interact with a line that has already been drawn, hover the mouse over either end point of the line. Left click and drag to move the end point, or right click to do any of the following:

  1. Delete the line
  2. Edit Properties (Line Thickness, Arrow Position)
  3. Add or remove arrows to the endpoints

ShowPopupKeyboardInDisplays=TRUE

NOTE: This option is TRUE by default.

How to enable or disable the Tablet PC Input Panel keyboard in Windows 7.¶

Use this keyboard to type without an actual keyboard.

  1. Click Start and select Control Panel.
  2. From the Control Panel, double click the Programs and Features item.
  3. Click the Turn Windows features on or off item.
     
    Turn Windows Features On/Off

     
  4. In the list of Windows features, scroll down to the Tablet PC Components item. If this item is not checked, check it. Click OK. At this point, Windows will install the Tablet PC Components if they weren't already installed. It might be necessary to reboot your computer when they are done installing.
     
    Tablet PC Components

     
  5. After the Tablet PC Components have been installed, you must configure the Tablet PC Input Panel keyboard to run when Windows starts. Open the Windows Start Menu, find the Startup folder, right-click it, and select Explore.
     
    Explore Option

     
  6. In the window that comes up, right-click and select New > Shortcut.
  7. Now, you must select the application to run, which is called TabTip.exe (this is usually located at C:\Program Files\Common Files\microsoft shared\ink\TabTip.exe). Either enter this location or browse to it using the Browse… button. Click Next.
  8. Click Finish to create the shortcut which will start TabTip.exe when Windows starts.
  9. Now a virtual keyboard will appear when using either the ADTDigital or ADTDataEntry display objects to enter a new value for an iTest channel.
     
    Virtual Keyboard
